# Petopia 实现规格 · 手感/动效 · v0.3 · 配套 DESIGN.md

> 本文件是给开发（Codex CLI）与美术/音频看的**手感 / Juice / 动效方向实现规格**。目标：让每一个反馈"怎么动、多快、什么曲线、配什么音与触感"都可照做、可验收，无需猜测。
> 规则、数值、系统状态机以 [DESIGN.md](DESIGN.md)（v0.3）为准；界面/控件/流程以 [spec-ux.md](spec-ux.md) 为准；美术风格以 [spec-art-overview.md](spec-art-overview.md) 为准。本文件只把它们**落成时序、曲线、粒子、音效、触感**。
> 音效命名引用 `spec-audio.md`（`[待细化]`：该文件尚未建立，本文用 `sfx_*` 占位命名并给出音色描述，供音频对齐）。
> 标记：`[待细化]` = 结构已定、细节留待补齐；`[待验证]` = 数值需 playtest 校准。
>
> **最高红线（继承 DESIGN §0.3 / spec-ux §7）**：治愈、零焦虑。任何手感不得制造惊吓、催促、损失感。冲突时，本文件的「零焦虑手感红线」（§5）优先于一切"更带感"的诉求。

# 0. 术语与全局参数

## 0.1 时长档（Duration Tokens）

统一时长命名，全项目复用。所有动效时长必须从此表取值，禁止散落魔法数字。

| Token | 时长(ms) | 用途 |
|---|---|---|
| `dur_instant` | 80 | 按钮按下形变、最小可感反馈 |
| `dur_fast` | 160 | 弹起、涟漪起步、小图标反应 |
| `dur_base` | 260 | 通用微交互主帧（摸头反应、Toast 滑入） |
| `dur_wash` | 400 | `WashTransition` 页面转场（spec-ux §0.2 400±100） |
| `dur_action` | 2000–4000 | 照料动作锁（DESIGN §3.5 INTERACT 2–4s） |
| `dur_show_s` | 3500 | 短演出（换模） |
| `dur_show_m` | 6000 | 中演出（回访入场、特殊事件） |
| `dur_show_l` | 12000–16000 | 长演出（毕业典礼，可点「继续」推进，不强制等待） |

> 治愈风的动效**偏慢**：微交互主帧 `dur_base`=260ms 比一般手游（120–180ms）刻意慢一档，配合「一顿一顿」（§1.3）营造松弛感。但**响应起步必须快**（`dur_instant` 80ms 内给出按下反馈），慢的是"回味"，不是"响应延迟"。

## 0.2 帧率与节拍

- 序列帧动画：**8–12fps**（spec-art §2.3，追求笨拙感）；骨骼动画在引擎内可跑 60fps 但**关键姿态之间刻意留停顿**（§1.3）。
- 目标渲染帧率：60fps（高端）/ 30fps（低端降级，§6）。
- 「呼吸节拍」基准：`breath_period` = 2800ms（一次完整吸-呼），idle 全局共享此节拍以求画面协调。

## 0.3 Haptic 分级（iOS `HapticFeedback` / Android 振动对齐）

零焦虑要求触感**只做"轻点头"，绝不做"警报"**。全项目仅允许以下 4 档，禁用连续/强/长振动。

| 档 | Flutter API | 体感 | 用途 |
|---|---|---|---|
| `haptic_none` | — | 无 | 冷却态点击、失败态、纯待机彩蛋 |
| `haptic_light` | `HapticFeedback.selectionClick()` | 极轻一下 | 按钮按下、Toast、切页签 |
| `haptic_soft` | `HapticFeedback.lightImpact()` | 柔软一顿 | 摸头/喂食成功、领取暖绒、盖邮戳 |
| `haptic_warm` | `lightImpact()` ×2 间隔 90ms（自定义柔和双击）`[待验证]` | 温暖两下 | 升级、换模、毕业收下、解锁图鉴 |

> **红线**：禁止 `heavyImpact` / `mediumImpact` 连发、禁止 `vibrate()` 长振。触感总能量随事件情感强度递增，但峰值封顶在 `haptic_warm`（两次 light）。设置页可全关触感（继承 spec-ux §0.4「音效可全关」同理扩展）`[待细化：设置项]`。

## 0.4 音效命名占位（引用 spec-audio.md `[待细化]`）

音频尚无独立 spec，本文用 `sfx_<场景>_<细节>` 占位并给音色描述。总红线（DESIGN §11.2 / spec-ux §0.4）：**无尖锐提示音、无突然大音量、可全关**；所有音效做 **20ms 淡入 / 60ms 淡出**避免爆音；峰值电平统一 **-12 LUFS `[待验证]`**，事件峰值不超过 idle 环境音 +6dB。

# 1. 动效原则

## 1.1 全局动画性格（Motion Personality）

四个词，任何动效评审对照：

1. **笨拙可爱（Clumsy-Cute）**：物体运动带一点点"没控制好"——弹入略微过冲再回弹、落地有软软的一顿、转身会多晃半下。不追求机械精确。对应 spec-art「颜色出线 1–2px、笔触抖动」的运动版。
2. **水彩晕染（Watercolor Bloom）**：出现/消失/转场优先用"晕开—化开"而非"平移—淡出"。颜色像滴进水里一样扩散（径向 mask + 边缘羽化），而非硬切。
3. **有呼吸感（Breathing）**：静止画面也在极轻微地动（§4）。世界"活着"，不是暂停的插画。
4. **柔和无惊吓（Gentle, No Startle）**：无高频闪烁、无骤然放大、无强震。加速度曲线平缓，峰值受控（见 §5 红线）。

## 1.2 缓动曲线库（Easing Library）

全项目 easing 从此库取，禁止任意手搓贝塞尔。Flutter 端给出 `Curves` 或 `Cubic(...)` 对照。

| Token | 曲线 | Flutter 对照 | 用途 | 典型时长 |
|---|---|---|---|---|
| `ease_pop_in` | easeOutBack（过冲回弹） | `Cubic(0.34, 1.56, 0.64, 1.0)` | 弹入：宠物入住、Toast、贴纸落下、图鉴收录 | `dur_base`~`dur_fast` |
| `ease_settle` | easeOutCubic（快出慢停） | `Curves.easeOutCubic` | 元素归位、卡片落定、页签翘起 | `dur_base` |
| `ease_press` | easeInOutQuad（对称柔和） | `Curves.easeInOut` | 按钮按压形变来回 | `dur_instant` |
| `ease_breath` | easeInOutSine（正弦呼吸） | `Curves.easeInOutSine` | idle 呼吸、柔光脉冲、经验条流动 | `breath_period` |
| `ease_wash` | 自定义"晕开"（先快扩散后缓沉） | `Cubic(0.4, 0.0, 0.2, 1.0)` | `WashTransition`、换模晕染、事件卡浮现 | `dur_wash` |
| `ease_drift` | 近线性微加减速 | `Cubic(0.45, 0.05, 0.55, 0.95)` | 云 / 视差 / 落叶飘 / 粒子漂 | 循环 |
| `ease_fade` | easeInOutQuad | `Curves.easeInOut` | 纯透明度淡入淡出、滤镜切换 | `dur_base` |
| `ease_wobble` | 阻尼弹簧（低刚度高阻尼） | `SpringDescription(mass:1, stiffness:120, damping:14)` | 笨拙晃动：尾巴、耳朵、名牌木牌摆动 | 依物理 |

**参数注记**：
- `ease_pop_in` 的过冲量控制在 **≤8%**（`1.56` 的 overshoot），可爱但不夸张；情感强度高的场合（毕业收下）可到 12%。
- `ease_breath` 幅度极小（缩放 ±1.5%，位移 ±2px），肉眼"感觉在动"但不干扰阅读。
- `ease_wobble` 阻尼偏高（damping 14），晃 1–2 下就停，避免"果冻感"太强显廉价。

## 1.3 「治愈系刻意的一顿一顿感」时序原则（Deliberate Stagger）

治愈手感的灵魂不是"流畅顺滑"，而是**恰到好处的停顿**——像手绘定格、像人放松时的动作。硬规则：

1. **关键姿态之间插入 `hold` 停顿**：一个动作 = 起 → **停(hold 80–160ms)** → 落 → **停** → 归位。停顿让笨拙感"生效"。例：摸头反应 = 抬头(120ms) → hold(120ms) → 眯眼蹭手(160ms) → hold(100ms) → 回落(160ms)。
2. **多元素入场错峰（stagger）**：同时出现的多个元素依次进场，**间隔 60–100ms**，不齐刷刷弹出。例：暖绒结算明细逐行落下、图鉴多格逐个显色、访客礼物逐件跳出。
3. **序列帧宁少勿满**：8–10fps 的"跳格"本身就是风格，不要补间到 60fps 丝滑（那会失去手账味）。
4. **反预期的小延迟**：偶发动作前留一个"迟疑"（如宠物看你 200ms 才决定蹭过来），比即时响应更"像活的"。仅用于**非功能性**反馈，功能性响应（按钮）仍须 80ms 内起步。
5. **收尾要"软着陆"**：任何动画结尾用 `ease_settle` / `ease_wobble`，末速趋 0，不硬停。

> 一句话：**响应快，回味慢，姿态之间敢留白。**

# 4. "活着的感觉"细节（Aliveness）

> 低成本高回报。这些让画面"活"，是治愈手感的地基。全部**极轻微**，不干扰阅读、不制造干扰。

## 4.1 宠物待机（Idle）微动
- **呼吸**：身体缩放 ±1.5% 沿 `ease_breath`，周期 `breath_period`(2800ms)；睡姿时呼吸更慢更深（周期 ×1.4，幅度 ±2.5%）。
- **眨眼**：随机间隔 **2.5–5s** 眨一次（2 帧快闭合，120ms）；偶发（10%）双眨。慵懒/睡眠态减少眨眼。
- **微动**：耳朵/尾巴 `ease_wobble` 极轻摆动，随机相位，避免整齐划一。
- **性格待机差异**：贪吃偶尔咽口水看饭碗；慵懒频繁打哈欠花式睡；好奇随机歪头；黏人镜头靠近时转向镜头（DESIGN §2.3）。

## 4.2 随机小动作触发节奏（Ambient Fidget）
- **触发时钟**：idle 状态下每 **8–15s** 随机抽一个"小动作"（无经验、`haptic_none` 或 `haptic_light`）：伸懒腰、挠痒、看天、追影子、蹭地、抖毛。
- **权重按性格**（复用 DESIGN §2.3 动作库）：活力→跑跳类权重↑；慵懒→睡姿类↑；爱幻想→望天/追影↑。
- **反预期迟疑**（§1.3-4）：小动作前 30% 概率先"迟疑"200ms 再做，增强"像活的"。
- **节流**：小动作与事件/互动演出**互斥排队**，不叠加轰炸；玩家操作时暂停 fidget。

## 4.3 环境响应（Environment Response，随 timeCtx 弱同步真实时间）
- **时段光影**：晨/午/黄昏/夜的环境色温缓慢过渡（跨时段用 `ease_fade` 长过渡 ~3s，**不刺眼切换**，DESIGN §11.2 / spec-art §1）。夜景由豪华度⑤+主题决定明暗。
- **天气微动**：晴→漂浮花瓣/柳絮；雨→雨丝 `ease_drift` + 地面小水洼倒影微波；雪→雪粒缓落；雾→半透白纱层缓移。全部**低密度、柔和**。
- **植被**：豪华度③+ 的树随"风"轻摆（`ease_wobble` 长周期）、随季节换色（换色用长 `ease_fade`）；花坛的花偶尔轻晃。
- **访客与环境互动**：萤火虫群夜间发光呼吸；水碗有鸟时水面微波纹。

## 4.4 镜头与视差（Camera & Parallax）
- **静态视差**：场景分层（spec-ux §3.1 z0–z6）做**轻微视差**——设备陀螺仪微倾时，远景（天空）与近景（宠物/前景植被）以不同速率位移（远慢近快，幅度 ≤6px），营造纸片层叠的手账立体感。低端机关闭（§6）。
- **呼吸式镜头**：主屏镜头极缓慢地"呼吸"（缩放 ±0.5%，周期 ~8s `ease_breath`），画面永不完全静止。
- **演出镜头**：毕业目送帧缓推近（§3.1）；换模时镜头微聚焦宠物（轻微 push-in 3%）。**禁**骤然拉近/甩镜（§5）。
- **交互反馈镜头**：点击宠物时镜头**不动**（避免眩晕）；仅在毕业等强情感演出用受控镜头运动。

> 成本提示：呼吸/眨眼/微动/fidget 复用少量骨骼动画 + 随机时钟即可实现，无需大量新美术；视差是纯代码分层位移。这些是"高回报低成本"的活性来源。

# 5. 零焦虑手感红线（Anti-Anxiety Motion Redlines）

> 继承 DESIGN §0.3 支柱、spec-ux §7、spec-art §1 情绪红线，并把它们落成**手感层可检查的硬约束**。评审逐条对照。**冲突时本节优先于任何"更带感"的诉求。**

| ❌ 禁止的手感 | 为什么 | ✅ 替代方案 |
|---|---|---|
| 屏幕猛烈震动 / screen shake | 惊吓、焦虑 | 柔光脉冲（`ease_breath` 亮度呼吸）或极轻的 `ease_wobble` 元素晃动；haptic 最高 `haptic_warm`（两次 light） |
| 高频闪烁 / 频闪 / stroboscope | 惊吓、可能诱发不适 | 柔和的亮度呼吸脉冲（周期 ≥1.5s，`ease_breath`）；星光/流星用低频柔亮 |
| 红光 / 红色警示 / 红点角标 | 警报感、焦虑（spec-ux §7.2） | 邮箱旗立起+暖色柔光晕（`MailboxBadge`）；进度用水彩描边非红条 |
| 突然的大音量 / 尖锐提示音 | 惊吓 | 所有音 20ms 淡入、峰值封顶（§0.4）；柔和铃铛/翻纸/水彩"噗"；可全关 |
| 催促抖动（元素抖动催点击）/ 弹跳催单 | 制造紧迫、催促 | 静态柔光呼吸提示；从不催促，等待即可 |
| 精确倒计时红字（如"14:59"） | 焦虑（spec-ux §7.1） | `WatercolorHourglass` 沙粒缓降 + 置灰；长按给"还在回味呢"柔和气泡 |
| 骤然放大 / jump-scare 式弹出 | 惊吓 | `WashTransition` 晕染浮现；`ease_pop_in` 过冲≤8% 温和弹入 |
| 硬弹窗 / 模态阻塞 Alert | 打断、焦虑（spec-ux §7.5） | `PawConfirm` 软纸卡柔和落下 |
| 连续多弹窗轰炸 | 信息过载 | 事件/演出**排队逐个**播放，不并发（§2.4 / §3） |
| "错过收益/损失/召回"类动效或文案 | 违反"离线只有收益"（DESIGN §3.2 / spec-ux §7.3） | 离线暖场动画"睡饱了蹭蹭你 +N"，只呈现正向 |
| 强对比硬阴影 / 硬切黑场转场 | 与手绘水彩柔和调性冲突（spec-art §1） | 柔和低对比光影；`ease_wash` 晕染转场无黑场 |

**峰值总封顶**（可测）：任何演出的 haptic ≤ `haptic_warm`；任何音效峰值 ≤ idle 环境音 +6dB；任何闪烁/脉冲频率 ≤ 0.67Hz（周期 ≥1.5s）；任何镜头缩放变化率 ≤ 1%/帧（无甩镜）；任何元素单帧位移 ≤ 屏宽 20% `[待验证]`。

# 6. 性能友好（Performance Degradation）

> 与后续 `spec-performance.md` 对齐 `[待细化]`。目标：中低端机流畅（30fps 不掉帧、不发烫），高端机 60fps 全特效。以下为降级策略占位，具体机型阈值/检测方式 `[待细化]`。

## 6.1 分级（Quality Tiers）`[待细化]`
- **High（高端）**：60fps、全粒子、视差开、骨骼动画、全环境微动。
- **Mid（中端）**：60fps、粒子数 ×0.6、视差开、微动全开。
- **Low（低端）**：30fps、粒子数 ×0.3 或关闭非必要粒子、**视差关**、`breath`/`fidget` 保留（低成本）、序列帧优先于骨骼、环境天气粒子密度 ×0.4。
- 分级检测：设备性能探测 / 掉帧自适应降级 `[待细化]`。

## 6.2 降级手段占位 `[待细化]`
- **减粒子**：粒子上限按 tier 钳制（爱心/星屑/雪雨密度）；超预算走对象池复用不新建。
- **降帧**：Low tier 场景层 30fps；UI 层保持 60fps 保证触摸跟手。
- **降视差/镜头呼吸**：Low tier 关闭陀螺仪视差与镜头呼吸（纯静态背景）。
- **演出简化**：Low tier 换模/毕业保留核心晕染帧，削减环境粒子与彩带密度（**不削减情感关键帧**）。
- **音频**：Low tier 关闭环境音层，保留交互音效。
- **发热保护**：连续高负载时自动降 tier `[待细化]`。

> 红线：降级只降"华丽度"，**绝不降"零焦虑"**（沙漏/柔光/无红点等红线在所有 tier 恒定生效），也不削减情感关键帧（毕业目送、换模凝出）。

# 7. 验收标准（Acceptance）

## 7.1 手感通用验收（可测条目）

- [ ] 所有动效时长取自 §0.1 时长档，无散落魔法数字。
- [ ] 所有 easing 取自 §1.2 曲线库，无任意手搓贝塞尔。
- [ ] 微交互响应起步 ≤ 80ms（`dur_instant`）；"回味"可慢（`dur_base`+）。
- [ ] 每个 §2 表中的核心交互都有完整反馈栈（动画+曲线+音效+haptic），缺项即不通过。
- [ ] Idle 状态可见呼吸（±1.5%）、随机眨眼（2.5–5s）、随机 fidget（8–15s），画面永不完全静止（§4）。
- [ ] 多演出同上线按 换模>毕业>事件>回访 排队，不并发（§3 / spec-ux §2.3）。
- [ ] 一顿一顿：关键姿态间有 hold 停顿、多元素错峰入场（§1.3），非齐刷刷补间。

## 7.2 零焦虑手感红线验收（发布前逐条勾选，继承 spec-ux §7.7）

- [ ] 全 App 无屏幕猛震、无高频闪烁、无红光/红点、无突然大音量、无催促抖动。
- [ ] haptic 全程 ≤ `haptic_warm`（两次 light），无 heavy/连振。
- [ ] 冷却仅以 `WatercolorHourglass` 沙粒缓降 + 置灰表现，无红字倒计时。
- [ ] 所有转场 `WashTransition` 晕染、无黑场硬切；所有确认软纸卡、无系统 Alert。
- [ ] 离线/毕业/收集反馈无任何"错过/损失/召回/失去"动效或文案。
- [ ] 任何脉冲/闪烁频率 ≤ 0.67Hz，任何镜头缩放率 ≤ 1%/帧（§5 峰值封顶）。
- [ ] 触感/音效可在设置页全关，关后游戏内进度与柔光提示照常（spec-ux §6.3）。

## 7.3 演出验收

- [ ] 毕业典礼（§3.1）：分段可点「继续」不卡死、结算=DESIGN §4.2、基调"旅程"无"失去"字眼、无闪光强震。
- [ ] 换模（§3.2）：旧形态晕开→新形态凝出可见、徽章纹样切换、Lv10 接毕业。
- [ ] 回访（§3.3）：铃铛+晕染显形→专属位、不占 slot 不吃冷却、摸头每日1次无红字。
- [ ] 收信（§3.4）：邮箱旗柔光非红点、不打断操作、回看模式无演出。
- [ ] 特殊事件（§3.5）：环境响应铺垫+插画+反应+结算骨架完整；彩蛋事件（篝火/彩虹）静默播线索计数+1。

## 7.4 MVP 子集（对齐 DESIGN §12 / spec-art §3）

MVP 必须实现的手感范围（其余 `[待细化]` 可后置）：

- **微交互（§2）**：点击宠物/摸头、喂食（普通+特殊）、玩玩具、洗澡、待机彩蛋；TapeButton 按压、WashTransition、WatercolorHourglass 冷却、PawConfirm、SoftToast；升级、收信、访客出现+互动、暖绒入账、购买、解锁收录、日常事件卡浮现、离线暖场。
- **演出（§3）**：毕业典礼（§3.1）、换模 Lv5/8/10（§3.2）、回访入场（§3.3）、收信小演出（§3.4）、特殊事件通用骨架 + 初雪/流星雨/满月三例分镜（§3.5）——对齐 DESIGN §12「Lv5/8/10 三次换模 + 毕业 + ≥5 明信片 + 1 回访」全闭环。
- **活着感（§4）**：呼吸、眨眼、随机 fidget、时段光影过渡、基础天气微动**必须进 MVP**（低成本、定义治愈手感基调）；陀螺仪视差可后置 `[待细化]`。
- **红线（§5）**：**全部红线在 MVP 即生效，无豁免**。
- **性能（§6）**：MVP 至少提供 High/Low 两档手动切换占位 `[待细化]`；自适应降级可后置。
