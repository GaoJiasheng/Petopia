# 0. 产品概述与设计支柱

## 0.1 一句话概念

> 一个手绘水彩的小院子，一次只养一只宠物。陪它从幼崽长大，满级后它「毕业」去环游世界——不是告别：它会持续寄回明信片，偶尔回院子串门。送走一只，再迎来下一只。

## 0.2 基本盘

| 项目 | 内容 |
|---|---|
| 平台 | iOS + Android（Flutter + Flame） |
| 联网 | **纯本地存储，无后端**，可完全离线游玩 |
| 玩法类型 | 单人 · 放置 + 轻互动 · 收集 |
| 美术 | 2D 手绘水彩、暖色、笨拙可爱、纸质手账 UI |
| 单局节奏 | 每日 2–4 次、每次 2–5 分钟的轻互动 |
| 单只宠物周期 | 约 2–3 周毕业（不加速） |

## 0.3 设计支柱（所有设计决策的最终仲裁标准）

1. **治愈，零焦虑**：没有饥饿死亡、没有心情惩罚、没有「你三天没来它很伤心」。离线只有收益，绝无损失。
2. **离别是旅程，不是失去**：毕业 = 关系形态改变（明信片 + 回访），情感账户只增不减。
3. **一次只爱一只**：单宠位强制专注，让每只宠物都被记住。
4. **收集是回忆，不是数值**：图鉴、相册、明信片都是「发生过的故事」的容器。
5. **轻手感、重细节**：不堆系统深度，堆动作、插画、文案的密度。

## 0.4 核心循环

```
┌─────────────────────────────────────────────────────┐
│  领养幼崽（随机 2 性格）                              │
│    ↓                                                 │
│  日常照料（喂食/摸头/玩耍/洗澡，带冷却）＋离线成长     │
│    ↓            ↖ 随机事件 / 访客来访 / 毕业宠回访    │
│  升级（Lv1→10，4 档成长模型）                         │
│    ↓                                                 │
│  Lv10 毕业 → 结算「暖绒」→ 出发旅行                   │
│    ↓                                                 │
│  收明信片（1–3 天一张）＋ 7–14 天随机回访             │
│    ↓                                                 │
│  领养下一只（院子豪华度 +1 档进度，解锁推进）→ 循环    │
└─────────────────────────────────────────────────────┘
```

## 1.2 图鉴 / 选择界面 UI 规则

宠物图鉴采用「手账贴纸册」形式，每种宠物一格。**每格必须处于以下四种状态之一**：

| 状态 | 视觉表现 | 文案规则 |
|---|---|---|
| `OWNED_BEFORE` 养过 | 彩色水彩画 + 毕业徽章 | 显示历任该物种宠物名字列表 |
| `AVAILABLE` 可养 | 彩色水彩画 + 「领养」按钮 | 正常介绍文案 |
| `LOCKED_KNOWN` 未解锁·条件已知 | **灰色铅笔线稿剪影 + 锁形贴纸** | **直接写明条件**，如「再送 1 只宠物毕业就能遇见它」，并显示进度条（2/3） |
| `LOCKED_HIDDEN` 未解锁·隐藏条件 | **问号形状的模糊水彩渍 + 「？」贴纸** | 显示**模糊线索**（1.1 表中的斜体句），线索随「访客前置达成」而逐步变清晰（两段式：未见过访客 → 只显示「？？？」；见过访客 → 显示线索句） |

> 设计意图：占位格明确传达「后面可以养这个」，制造期待而非焦虑；隐藏彩蛋给谜语不给答案。

## 1.3 领养流程状态机

```
[图鉴选择物种] → [随机生成个体]
   个体 = 物种 × 模型变体(5选1，加权随机) × 性格标签(10选2，见§2)
→ [取名（可重掷变体1次/日）] → [幼崽入住 Lv1] → 养育期(§3)
→ [Lv10 毕业典礼事件] → 旅行期(§6) → 回访期(§7，永久)
约束：院子在养位 = 1；必须当前宠物毕业后才能领养下一只。
```

## 1.4 美术资产需求表（制作规格）

**规则 A：每种宠物 ≥ 5 个模型变体**（颜色/花纹/性格外观差异，如橘猫：全橘/橘白/奶牛橘/浅杏/深炭橘）。
**规则 B：不同性格对应不同动作效果，每个性格 ≥ 10 个动作**（动作 = 骨骼/序列帧小动画，如「贪吃」性格的猫有专属的舔盘子、盯饭碗、打饱嗝……）。

| 资产项 | 计算方式 | 数量（常规 8 种 + 彩蛋 4 种） |
|---|---|---|
| 基础模型变体 | 12 物种 × 5 变体 | 60 套变体皮肤 |
| 成长阶段模型 | 每变体 × 4 档（Lv1/5/8/10，见§3.6） | 240 个阶段形态（同变体可换色复用骨骼） |
| 通用基础动作 | 每物种 8 个（走/坐/睡/吃/被摸/洗澡/玩/待机） | 12 × 8 = 96 |
| 性格专属动作 | 10 性格 × 10 动作，按物种适配 | 100 个动作模板 × 物种适配层 `[待细化]` |
| 图鉴插画 | 每物种：彩色 1 + 线稿剪影 1 + 问号渍 1（彩蛋） | ≈ 28 张 |

> 实现建议：性格动作先做「动作模板库」（100 条），再按物种做骨骼适配，避免 12×100 全量手绘。

## 3.2 离线经验规则（renew 机制）

**文字规则**：离线每满 1 小时 +1 经验，单次结算封顶 12 点；**每次上线即结算并重置计时**——即离线收益按「上次在线时刻 → 本次上线时刻」的时长计算并封顶，然后从本次上线重新计时。一天内多次上线，各段分别结算，但受「自然日累计 12 点」总上限约束。

**伪逻辑**：

```pseudo
onAppResume(now):
    elapsedH = floor((now - lastOnlineAt) / 1h)
    gain     = min(elapsedH, 12)                       # 单段封顶
    gain     = min(gain, 12 - offlineExpGrantedToday)  # 自然日总上限
    if isLazy(pet): dailyCap = 13                      # 慵懒标签特例
    if gain > 0:
        addExp(gain, source=OFFLINE, note=f"{elapsedH}h offline")
        offlineExpGrantedToday += gain
    lastOnlineAt = now                                 # ★ renew：重置计时
    if crossedMidnight(now): offlineExpGrantedToday = 0
```

> 要点：`lastOnlineAt` 在**每次上线**都刷新，因此「上线一下再离线」会重新开始计时；离线 3 天也只按最后一段结算 + 每日上限，符合「零焦虑、不奖励爆肝也不惩罚离线」。UI 上以「宠物睡饱了，蹭蹭你 +N」的暖场动画呈现，不出现「错过收益」类文案。

# 4. 货币系统 ——「暖绒」（Warm Fluff）

## 4.1 命名与世界观

资源名为 **「暖绒」**：宠物在被好好爱着的时候，会掉落一种带着体温的、发着微光的绒毛——是「被陪伴过的时间」凝成的东西。图标：一小撮发光的水彩绒毛。命名理由：贴合「毛茸茸 + 温暖」核心体验，且天然解释产出逻辑（陪伴产生暖绒），避免金钱感。

## 4.2 产出规则

**毕业结算（主要来源）**：

```
毕业暖绒 = 基础 200
         + 事件参与数 × 2        （上限 +100）
         + 访客互动次数 × 3      （上限 +60）
         + 隐藏彩蛋：若为彩蛋宠物 +80
预期单只毕业产出 ≈ 260 ~ 380
```

**日常小额产出**：每日首次照料 +5；宠物升级 +10/级；成就奖励（见 §1.5）；回访宠带小礼物 +10~20。日常月均 ≈ 250 `[待验证]`。

## 4.4 加速平衡说明（保护 2–3 周基准）

原则：**加速道具只提升「主动动作单次收益」，不缩短冷却、不提高每日次数上限、不提高离线上限。**

- 理论加速上限：喂食 12 次全用特饼干（+36/天）+ 两玩具永久强化（+16/天）→ 日均 45 → 97。
- 但特殊食粮为**消耗品且暖绒供给有限**（月均 ~250 暖绒 ≈ 15 次特殊喂食），实际可持续日均 ≈ 60–65。
- 结果：极限玩家毕业时间 17–18 天 → **12–13 天**，仍 ≥ 大版本节奏红线（10 天）`[待验证]`；且经验曲线本身不变，审计日志中加成部分记为独立 `ITEM_BONUS` 条目，可追溯。
- **明确不做**：任何直接「+N 级」「跳过等待」类道具。

## 6.3 明信片生成模板（数据结构）

**生成公式**：`地点 × 时间(季节/时段/天气) × 遭遇(人/动物) × 事件 → 正文(性格文风渲染) + 照片插画 + 邮戳`

```
PostcardTemplate 生成管线:
1. location   = journey.nextStop
2. timeCtx    = {season, timeOfDay, weather} 随机 + 与真实日期弱同步
3. encounter  = weightedPick(location.encounterPool, pet.personality)
              // 遭遇 = 人物或动物 + 关系动词（帮助/被请客/比赛/迷路同行/交换礼物…）
4. incident   = weightedPick(incidentPool[location.vibe], pet.personality)
              // "碰撞"：趣味事故/巧合/小奇迹
5. bodyText   = render(templateBank[主性格], {location, timeCtx, encounter, incident, ownerName})
6. photoId    = f(location.photoStyle, pet.species, incident.poseHint)   // 见 6.5
7. 存入 postcardAlbum 与 travelAlbum（双入册，§6.6）
```

```
Postcard {
  id, petId, journeyId
  locationId, seq            // 第几站
  sentAt, receivedAt
  timeCtx: {season, timeOfDay, weather}
  encounterId, incidentId
  bodyText: string           // 渲染定稿（存文本，保证回看一致）
  photoAssetId: string
  stampId: string            // 邮戳 = 地点徽章，集邮子玩法
  cluesTo: petId? | visitorId?   // 可选：彩蛋解锁线索载体
}
```

# 7. 回访系统

## 7.1 规则

- 毕业宠物旅程结束后进入 **「世界漫游」** 状态，**每 7–14 天（随机）回院子串门 1 次，停留 1–2 天**。
- 回访**作为当日的一个事件**触发（进入 §9 事件队列，类型 `REVISIT`），上线时以「熟悉的铃铛声 + 邮箱旁的身影」演出。
- **同时最多 1 只回访**：若多只毕业宠命中同一窗口，按队列顺延（先到先访）。
- **不挤占在养位**：回访宠出现在院子专属位置（豪华度 ④ 后是落脚小屋，之前是篱笆边），不占用宠物在养 slot，不吃照料动作冷却。

## 7.2 调度伪逻辑

```pseudo
onGraduationRoamStart(pet):
    pet.nextRevisitAt = now + random(7d, 14d)

daily_tick():
    due = roamingPets.filter(p => p.nextRevisitAt <= today)
    if due and currentRevisitor == null:
        v = due.earliest()
        spawnEvent(REVISIT, v, stay = random(1d, 2d))
    # 其余 due 宠物保持排队，离开后重排

onRevisitEnd(pet):
    pet.nextRevisitAt = now + random(7d, 14d)
```

## 7.3 回访内容

| 模块 | 设计 |
|---|---|
| 与玩家 | 可摸头 1 次/天（+回忆动画，播放它幼年同款动作的「长大版」）；带小礼物（暖绒 10–20 / 装饰贴纸 / 特殊零食 ×1） |
| **与在养宠物的专属互动** | 每对「回访物种 × 在养物种」有专属事件文本+动画。示例：毕业橘猫 × 在养仓鼠 →「阿橘把尾巴圈成一个窝，豆豆理直气壮地睡了进去」；毕业柴犬 × 在养乌龟 →「大福非要教石头接飞盘，最后飞盘停在了龟壳上，双方都很满意」。在养宠物获 +5 经验（来源 REVISIT） `[待细化：12×12 矩阵，先做 20 对高频组合]` |

# 10. 数据模型总览（本地存储 Schema 蓝图）

> 建议：**Hive/Isar 存对象 + SQLite 存流水型大表**（ExpLog、Postcard）。所有表带 `schemaVersion` 支持迁移。

```
PetSpecies      { id, name, category(REAL|FANTASY), unlockRule{type, threshold?, clueText?, hiddenSteps?}, variants[5] }
Pet             { id, speciesId, variantId, name, personality[2]: tagId,
                  bornAt, level, exp, stage(A|B|C|D),
                  state(RAISING|TRAVELING|ROAMING|REVISITING),
                  lastOnlineAt, offlineExpGrantedToday,
                  wishId?, graduatedAt?, journeyId?, nextRevisitAt? }
PersonalityTag  { id, name, eventWeightMap{eventTag: mult}, actionSetId, postcardStyleId }
ExpLogEntry     { id, petId, timestamp, sourceType, sourceRef?, delta, levelAt, expAfter, note }   // §3.4，只增不改
CurrencyWallet  { balance }               // 暖绒
CurrencyLog     { id, timestamp, delta, reason, ref? }        // 同样可审计
YardState       { luxuryStage, activeThemeId, ownedThemeIds[], slots[{pos, itemId?}], foodTray{foodType?, placedAt} }
ShopItem        { id, category, price, effect{type, params}, artRef }
Journey         { id, petId, stops[locationId × 5–8], currentIdx, nextPostcardAt, state }
Location        { id, name, category, climate, vibeTags[], photoStyle, encounterPool[] }
Postcard        { …见 §6.3 }
Visitor         { id, name, rarity, activeTime, weatherPref, foodPref, seasonPref, artRef, clueRole? }
VisitorLogEntry { id, visitorId, date, interactionId?, withPetId }         // 来客图鉴数据源
VisitorPetInteraction { …见 §8.4 }
Event           { …见 §9.1 }
EventLogEntry   { id, eventId, petId, date, choiceIdx?, expGranted }
Achievement     { id, name, hidden: bool, clueText?, condition{type, target, count}, progress, unlockedAt?, reward }
ClueCounter     { clueId, count, threshold }                  // 彩蛋解锁进度（火光×3 等）
Albums          （视图层：由 Postcard / ExpLog / EventLog / VisitorLog 聚合，不单独存储）
Settings        { notifications, sound, schemaVersion, createdAt }
```

**关键不变量**：
1. `pet.exp == sum(ExpLogEntry.delta where petId)` —— 启动时校验，可审计的根保证。
2. 处于 `RAISING` 的 Pet 全局 ≤ 1；处于 `REVISITING` 的 ≤ 1。
3. `ExpLogEntry / CurrencyLog` 只追加，不更新不删除。

# 11. 技术与美术规格

## 11.1 技术选型：Flutter + Flame

| 层 | 模块 | 职责 |
|---|---|---|
| 数据层 | Isar/Hive + SQLite；`SaveService`（自动存档、双备份、schema 迁移）；`AuditService`（流水写入与校验） | 纯本地持久化 |
| 逻辑层 | `ClockService`（离线结算、防调表：单调时钟+合理性钳制 `[待细化]`）；`ExpEngine`；`EventScheduler`（事件/访客/回访统一日程队列）；`PostcardGenerator`；`UnlockService`（图鉴/成就/线索计数）；`EconomyService` | 全部规则集中于此层，可单测 |
| 表现层 | Flame：`YardScene`（分层：主题背景/豪华度布局/宠物 Spine 或序列帧/天气粒子）；Flutter：手账 UI（图鉴/相册/商店/成长手账）；`flutter_local_notifications`（明信片/回访提醒，可关闭） | |

其他：无任何网络权限申请；存档导出/导入（本地文件）作为「防丢失」方案 `[待细化]`。

## 11.2 美术风格规范

- 手绘水彩：留白纸纹、颜色出线 1–2px 的「笨拙感」、暖色主调（米黄底 #FAF3E3 系）。
- UI = 纸质手账：胶带贴纸按钮、铅笔手写体标题、图钉/回形针作页签。
- 动效：水彩晕染转场；禁止硬弹窗与红点轰炸（每日至多 1 个温和的邮箱提示）。
