# Petopia 美术资产规格 · UI 控件与图标（Art Spec · UI）

> **版本：v0.3** ｜ 配套 [spec-art-overview.md](spec-art-overview.md)（全局风格/技术/命名总纲） ｜ **本文件是【UI 美术资产】的权威来源**。
>
> 依据：[spec-ux.md](spec-ux.md)（16 屏清单 + 通用组件 + 零焦虑红线）、DESIGN.md §11.2（水彩手账 UI 规范）、§3.6（等级徽章 4 档）、§4（暖绒）、§1.2（图鉴四态）。
>
> **整体风格铁律**（继承总纲 §1）：纸质手账 + 奶油 / 可爱风——胶带贴纸按钮、手写体、图钉/回形针页签、水彩沙漏冷却态；线条出线 1–2px、水彩晕染、纸纹颗粒；形体圆润、低饱和、柔和可亲；**禁止**红点/红字倒计时/硬弹窗/刺眼闪光（spec-ux §7）。
>
> **尺寸单位**：均为 **@1x 逻辑 px**（参考画布 1080×1920 竖屏），正式交付全部为成品水彩渲染 **PNG @1x/@2x/@3x 三套**。SVG 仅可作为内部草稿/路径参考，不得作为进包交付物；图标、徽章、按钮同样按终稿 PNG 交付。
>
> **命名**：控件类五前缀 `ui_btn_X / ui_icon_X / ui_badge_X / ui_tex_X / ui_frame_X`；本清单按总纲 `<域>_<主体>_<变体>` 总格式扩展两个前缀：`ui_chip_X`（筛选/页签小件）与 `ui_fx_X`（转场/反馈特效），已与总纲 §2.4 兼容。
>
> **MVP 列**：★ = MVP 子集（详见 §7）。字段模板遵循总纲 §2.5。

# 2. 字体规格（视觉要求，不含字体文件）

> 对齐 spec-ux §0.1「标题=铅笔手写体；正文=圆润无衬线手写感」。此处只定视觉验收标准与字号档，选型/授权 `[待细化]`。所有文字色默认暖棕 `#6B5445`，**禁用纯黑与红色**。

### 2.1 标题体「铅笔手账体」
- **视觉**：铅笔/彩铅质感，笔画有轻微颗粒飞白与粗细抖动，字形略歪斜（±2°随机感），像主人在手账上认真写的标题。中文覆盖 GB2312 常用字即可，需含数字与基础标点。
- **用途**：屏幕标题、按钮文字、名牌名字、明信片落款。
- **字号档（@1x）**：H1 44 / H2 36 / 按钮 32 / 名牌 28。

### 2.2 正文体「圆润手写体」
- **视觉**：圆头笔触、x 高较大、字距宽松、无衬线但带手写弧度；长文阅读不累。事件文案、手账流水、明信片正文用它。
- **用途**：正文、气泡、Toast、条目、说明文字。
- **字号档（@1x）**：正文 28 / 次级 24 / 注释 20（最小可用 20，低于此不排文字）。

# 7. MVP 子集说明

> 对齐 DESIGN §12：MVP = 院子主屏 + 核心照料动作 + 图鉴（含全部占位格四态）+ 双相册 + 领养/换模/毕业全流程演出。上表 ★ 即 MVP 必做。要点：

- **优先级 P0（第一批交付）**：基础套件 22 项全量（★22）、S1 主屏 9 项、照料动作按钮与图标、`ui_icon_hourglass_wc`、`ui_frame_mailbox_flag`、等级徽章 4 档、`ui_fx_wash_transition`。——没有这批，主屏跑不起来。
- **P1（闭环所需）**：S4 图鉴四态件、S6 相册缩略/S8 查看器与收信演出件、S12 领养流程、S13/S16 演出 UI、暖绒/商店 MVP 3 分类（主题/装饰/食玩合并页签可后补第 5 类）、S15 引导。
- **可延后（非 MVP，无 ★）**：S3 水彩饼图、S7 旅行相册全套（毕业后 3 天内才需要，可随 P1 末尾补）、`ui_frame_journey_teaser`、邮戳进度条、书架、相册皮肤分类、导出导入按钮、`ui_badge_new_soft`、雪/风天气图标等。
  - 注意：S7 旅行相册在 MVP 验收路径内（毕业→建册），故 `ui_frame_book_cover`/`ui_frame_book_page` 虽未标 ★，**MVP 后期仍需最简版**（可先用 `ui_frame_card_sticker` 拼装占位）。
