## 0. 本域公共约定（一次声明，全文生效）

- **用途（默认值）**：院子主屏宠物本体动画 + 图鉴立绘底图 + 换模演出（EVOLVE）素材。个别条目另有用途时在行内标注。
- **格式（默认值）**：**Spine 骨骼动画（首选）**——每「物种 × 成长档」一套骨骼 + 部件切图；**同物种 5 个变体以 Spine skin（换色皮肤）复用同一套骨骼**，仅重绘部件贴图配色，不重做绑定。序列帧仅用于特效与个别难以骨骼化的演出（行内标注 `sprite sheet`）。
- **骨骼复用规则（省成本要点）**：
  1. **变体换色复用**：12 物种全部适用——5 变体共享骨骼与动作数据，只换 skin 贴图（花纹变化大的变体如奶牛橘/胡麻柴需重绘贴图但不改切片结构）。
  2. **D 旅装复用 C 成年**：D 档 = C 档骨骼 + 配件插槽（attachment slot：背包/围巾/帽子/风镜等），**不新做骨骼**。实际新绑定骨骼 = 12 物种 × 3 档（A/B/C）= 36 套，外加 12 组旅装配件贴图。
  3. **跨物种复用**：`pet_uni`（独角兔）直接复用 `pet_rabbit` 全部 3 套骨骼 + 角/鬃毛/柔光附件；`pet_cat` 与 `pet_shiba` 四足绑定结构互为参考（切片命名对齐，动作曲线可互相拷贝后调味）。
  4. **动作重定向**：通用动作与性格动作均以 **C 档为母版**制作，向 A/B 档重定向后做比例与「笨拙度」微调（A 档动作幅度小、频率低、常失误），D 档直接沿用 C 档并附加配件物理（围巾飘动等）。
- **帧数标注**：Spine 条目标「关键帧密度 ≈ 帧数 @ fps」等效值，统一按总纲 §2.3 治愈帧率 **8–12fps** 设计；`循环` = 无缝 loop，`单次` = one-shot。
- **画布尺寸（@1x 逻辑 px）**：默认 **512 × 512**（角色本体占画布约 70%，四周留动作与特效余量）。按物种体型微调（理由随各节标注）：
  | 物种 | 画布 | 理由 |
  |---|---|---|
  | 橘猫 / 柴犬 / 垂耳兔 / 乌龟 / 变色龙 / 小火龙 / 独角兔 | 512×512 | 标准体型 |
  | 仓鼠 | 384×384 | 体型最小，缩画布省内存，避免缩放糊边 |
  | 鹦鹉 | 448×512 | 纵向体型（站姿高、尾羽长） |
  | 玉米蛇 | 640×448 | 横向体型（S 形滑行需要横向余量） |
  | 小幽灵噗噗 | 448×448 | 团状悬浮体 + 半透明边缘留白 |
  | 星星虫 | 384×384 | 微型体 + 光晕留白（光晕单独出特效层，见 §4.4） |
  > 成长档不改画布：A 幼崽画得小（占画布 ~45%）、C/D 画得大（~70%），保证换模演出时锚点稳定不跳。
- **命名**：严格按总纲 §2.4——形态 `pet_<种>_varNN_stageX`；动作剪辑导出名 `pet_<种>_varNN_stageX_<动作>`；性格动作模板 `act_<性格缩写>_NN`，物种适配后加后缀（如 `act_glu_01_cat`）。物种短名对照：`cat / shiba / rabbit / hamster / turtle / parrot / snake / cham / ember / uni / boo / starbug`。
- **调色**：各变体主色组均为**水彩晕染基调色**（允许 ±10% 明度浮动与边缘渗色），全部从总纲 §1.1 全局调色板派生或与其和谐；线条统一暖棕 `#6B5445`。
- **关联（默认值）**：content-pets-personalities.md §A 对应物种节 + DESIGN §1.4/§3.6。

## 3. 性格专属动作模板（10 性格 × 10 = 100）

> **模板制作方式**：每条动作以「通用四足中型体」（以 cat C 档骨骼为基准）制作母版 `act_<性格>_NN`，再按物种做**骨骼重定向适配**，导出加物种后缀（如 `act_glu_01_shiba`）。适配三类：
> - **四足系**（cat/shiba/rabbit/hamster/uni）：直接重定向 + 比例修正；
> - **异构系**（turtle/parrot/snake/cham/starbug）：保留动作意图与节奏，肢体表达替换（如「递爪」→ 鹦鹉抬翅、蛇抬头颈、乌龟伸脖）——每条模板在异构物种上按「意图注释」二创 `[待细化：物种适配层，逐条注释见后续适配表]`；
> - **悬浮系**（boo）：位移类动作改为漂浮位移，触地类动作改为悬停模拟。
> **公共字段**：用途 = 性格系统待机插播 / 事件演出 / 互动替换动画（触发逻辑见 content §C）；尺寸 = 模板画布 512×512（适配后随物种画布）；格式 = Spine（叠加符号/粒子处标注 sprite）；变体 = ×适配物种数（完整版 12，MVP 3）；关联 = content §B 对应性格节 + DESIGN §1.4 规则 B。
> **帧数缺省值**：循环类 12帧@8fps、单次演出类 16–24帧@10fps，下表仅标注偏离缺省的条目与关键叠层。
