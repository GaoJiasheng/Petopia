# 0. 分层合成方案与全局约定

## 0.1 合成公式（DESIGN §6.5）

```
照片插画 = pc_bg_地点(40)
         + pc_pose_物种_姿态(12×8=96)   // 摆放于背景板锚点
         + pc_filter_天气(6)             // 半透明整层覆盖
         + pc_sticker_碰撞(60)           // 小道具/配角点缀
邮戳     = pc_stamp_地点(40)             // 盖在卡纸 chrome 的邮票位上
特例     = pc_special_X(≈20)             // 整幅替换「照片插画」层组
```

图层顺序（自下而上）：`bg → pose → sticker(可在 pose 前后各 1 层，见贴纸表「层位」) → filter → chrome 相框`。

## 0.2 照片画布与锚点位约定

- **照片画布**：`1080 × 720 @1x`（3:2 横构图；明信片是本项目唯一横构图域，卡纸 chrome 负责与 9:16 主画布的排版衔接）。
- **锚点位（anchor）**：每张背景板标定 **2–3 个宠物锚点**，坐标为「姿态贴层**底部中点**的对齐点」，格式 `(x, y, scale)`，坐标系 = 照片画布左上角为原点。scale 为姿态贴层的建议缩放（营造近大远小）。
- **锚点朝向**：每个锚点标注 `→左 / →右 / →深处 / →镜头`，姿态贴层默认朝左绘制，水平翻转适配（不对称物种如变色龙卷尾方向允许翻转）。
- **四种标准锚点布局**（背景板表内引用，个别地点带覆盖值）：

| 布局 | 适用 | 锚点 A（主位） | 锚点 B | 锚点 C |
|---|---|---|---|---|
| **L1 前景主位型** | 开阔场景、合影/吃/睡 | (540, 600, 1.0) →镜头 | (300, 560, 0.85) →右 | (780, 480, 0.7) →左 |
| **L2 远眺型** | 眺望类构图（云海/星空/极光） | (400, 620, 1.0) →深处 | (700, 600, 0.9) →深处 | — |
| **L3 街景纵深型** | 街道/巷道/桥的透视场景 | (360, 640, 1.0) →右 | (640, 540, 0.75) →深处 | (860, 470, 0.55) →深处 |
| **L4 室内/近景型** | 屋内、摊位、工坊 | (540, 580, 1.1) →镜头 | (260, 600, 0.9) →右 | — |

- **锚点与 poseHint 的匹配**：`眺望` 优先 L2/A；`合影` 优先 L1/A；`泡水` 锚点 y 即水面线（贴层水下部分被背景水面遮罩裁切）`[待验证：遮罩交付方式——建议背景板附一张同尺寸水面 alpha 遮罩]`。
- **格式**：本域全部为 **PNG（透明通道）**，水彩栅格 + 纸纹颗粒；导出 @1x/@2x/@3x 三套（总纲 §2.1–2.2）。

## 0.3 风格红线（承总纲 §1）

- 背景板是「照片感水彩」：比院子场景更「取景框」——留白多、主体明确、允许轻微暗角与水彩纸白边呼吸感。
- 一切滤镜/贴纸不得引入高饱和赛博色与硬阴影；惊讶等情绪符号用手绘抖线，不用漫画速度线的锐利版本。
