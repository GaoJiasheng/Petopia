# 0. 适配设计原则

## 0.1 三条不可让步的硬红线（优先级从高到低）

1. **全屏满幅、绝不黑边（第一优先级）**：任何设备、任何长宽比下，背景/场景层必须填满整个物理屏幕（含刘海/挖孔区域与圆角外沿），**绝不允许出现 letterbox（上下黑边）或 pillarbox（左右黑边）**。
2. **绝不拉伸变形**：填满不得靠非等比拉伸实现。宠物、UI 手账元素、水彩纹理的**长宽比恒定**；填满靠「等比放大到覆盖 + 边缘延展/裁切溢出区」实现，而非把画面横向或纵向拉长。
3. **核心内容绝不被遮挡或裁切**：所有可读文字、可点控件、宠物主体、情感关键帧必须完整落在 **safe-area 内容区**，绝不被刘海/灵动岛/挖孔/手势条/圆角吃掉。

> 冲突仲裁：红线 1（填满）与红线 3（不遮挡）看似矛盾——解法是**分层**：背景/场景层 full-bleed 到物理屏（红线 1），交互与关键内容层锚定在 safe-area 内（红线 3），中间用 overscan 溢出区吸收比例差异（见 §3）。

## 0.2 核心机制一句话

> **背景满屏铺（cover + overscan），内容缩在安全框（safe-area contain），比例差异由可裁切的溢出余量吃掉——永不黑边、永不变形、永不遮挡。**

# 2. 长宽比范围与安全区表

## 2.1 需覆盖的长宽比跨度

> **纵横比设计跨度：`4:3`（最方，大平板/折叠内屏）～ `21:9`（最长，超长屏/折叠外屏）。** 所有 full-bleed 资产必须在这个跨度内任意比例都填满且不变形。

| 极端 | 比值(高:宽) | 出现场景 | 主导溢出方向 |
|---|---|---|---|
| 最「长」 | 21:9 ≈ 2.33 | 索尼超长屏、折叠外屏 | 竖向内容不够 → **需垂直 overscan 余量** |
| 主流 | 19.5:9 ≈ 2.17 | iPhone 长屏、主流安卓 | 设计基准附近，溢出最小 |
| 较「方」 | 16:10 ≈ 1.60 | 平板、折叠内屏 | 横向偏窄 → 横向裁切 + **需水平 overscan 余量** |
| 最「方」 | 4:3 ≈ 1.33 | 大平板、折叠内屏 | 横向最窄 → **需最大水平 overscan 余量** |

**结论（给美术的硬指标）**：竖屏主背景以 `1290×2796`（≈19.5:9）为基准绘制，四周各留**可裁切安全溢出余量**：
- **上下各 ≥ 12%**（吸收到 21:9 的额外高度需求）。
- **左右各 ≥ 15%**（吸收到 4:3 的额外宽度需求）。
- 详见 §3.2 与 §6。

## 2.2 安全区（safe-area inset）规则

> Flutter 统一走 `MediaQuery.of(context).padding`（= viewPadding 减去被键盘等占用部分）与 `viewPadding`。Flame 侧需把 safe-area 作为**内容锚定框**传入。inset 单位为逻辑 pt/dp。

| 区域 | 来源 | 规则 | 数值参考 |
|---|---|---|---|
| 顶部·刘海 | `padding.top` | 顶栏（名牌/经验条/暖绒）下沿 ≥ padding.top + 8 | iPhone 刘海 ~47–50 |
| 顶部·灵动岛 | `padding.top` | 同上；灵动岛比刘海更高更靠中，顶栏元素**避开中央岛区**，左右分置（名牌左、暖绒右，中间留空） | ~59 |
| 顶部·挖孔(Android) | `WindowInsets` cutout | 顶栏避开挖孔矩形；居中挖孔时中央不放可点元素 | 24–40 `[待验证]` |
| 底部·手势条 | `padding.bottom` | 底栏 `ClipTab` 与动作条下沿 ≥ padding.bottom；底栏背景可延伸到屏底（full-bleed），但**可点区**上移到手势条之上 | iOS 34；Android 手势 16–48 |
| 屏幕圆角 | 常量估计 | 四角 ≥ 16pt 半径内不放关键内容/文字；圆角处只走背景延展 | 圆角 R≈44–55px `[待验证]` |
| 侧边·曲面屏(类 C) | 厂商 | 左右各额外内缩 ≥ 8dp，避免曲面畸变区放控件 | `[待验证]` |

### 2.2.1 内容安全框定义（供代码/美术共用）

```
safeContentRect = 物理屏
  ├ 顶部内缩 = max(padding.top, cutout.top) + 8
  ├ 底部内缩 = padding.bottom + 8
  ├ 左内缩   = max(padding.left, 圆角内缩16, 曲面内缩)
  └ 右内缩   = max(padding.right, 圆角内缩16, 曲面内缩)
```
- 所有 UI 一级容器（顶栏、底栏、动作条、弹层、明信片查看器）以 `safeContentRect` 为父框布局。
- 背景/场景层**不**受此框约束（它铺满物理屏）。

# 3. 全屏满幅策略

## 3.1 分层填充模型（对齐 spec-ux §3.1 Flame z 层）

| 层 | 填充策略 | 依据 |
|---|---|---|
| z0 主题背景 / z1 豪华度布局 / z6 天气粒子 | **full-bleed cover**：等比缩放到「宽高都 ≥ 物理屏」，多出部分溢出屏外被裁 | 红线 1 填满 |
| z2 可放置物 / z4 宠物 / z5 访客 | 锚定在**场景锚点**（相对背景的百分比坐标），随背景一起等比缩放；主体保证落在 safe-area 内 | 红线 2 不变形 + 红线 3 不遮挡 |
| z3 回访宠层 | 同 z4，专属锚点（篱笆边/落脚小屋）按百分比定位 | — |
| z7 UI 覆盖层 | **contain in safeContentRect**：锚定在安全框，绝不进入 overscan 溢出区 | 红线 3 |

## 3.2 背景 full-bleed + overscan 具体做法

**缩放公式（cover）**：
```
scale = max(screenW / artW, screenH / artH)   // 取 max = 覆盖（cover），非 min（那会留黑边）
drawW = artW * scale ; drawH = artH * scale
offsetX = (screenW - drawW) / 2   // 居中，多余部分对称溢出屏外
offsetY = (screenH - drawH) / 2
```
- `max` 保证**至少一个维度铺满、另一维度溢出**，永不出现黑边。
- 美术交付的背景含 §2.1 的溢出余量（上下 12%、左右 15%）；`scale` 放大后被裁掉的正是这些**可裁切延展区**，核心构图仍完整。
- **锚定微调**：允许背景在 cover 后做 ≤ ±3% 的 `offsetY` 微调，把地平线/院子主构图对齐到 safe-area 中央（避免主体被顶栏/底栏压住）。`[待细化]` 每主题的锚点偏移值。

**禁止 letterbox 的实现约束**：
- Flame `camera` / `Viewport` **不得**使用 `FixedResolutionViewport` 默认的 letterbox 行为；必须用 **cover 型 viewport**（自定义或 `MaxViewport` + cover 缩放），确保填满。
- Flutter 图片一律 `BoxFit.cover`（**禁止 `contain`/`fitWidth` 单边**导致另一边留白）。
- 根 `Scaffold` 背景色即便设深色也只是兜底；正常运行任何一帧都不应露出该底色。

**极端比例的解法（延展/裁切而非黑边）**：
- **过长（21:9）**：竖向不够 → 背景上下的延展区（天空/草地留白）被拉入可视，天空多露一点、草地多露一点，构图仍成立；宠物锚点按百分比自然下移，仍在 safe-area 内。
- **过方（4:3 / 折叠内屏）**：横向不够 → 左右延展区（篱笆外的远景/花丛）被裁掉，中央院子主构图保留；UI 安全框相应变宽，控件横向留白增大但不拉伸。
- 任何比例都不改变资产宽高比，只改变「露出多少溢出区」。

## 3.3 核心内容锚定（safe-area 内响应式）

- UI 元素用**锚点 + 相对偏移**，不用绝对像素。锚点集：`topLeft / topCenter / topRight / bottomLeft / bottomCenter / bottomRight / center`。
- 顶栏：名牌锚 `topLeft`（safe 内），暖绒锚 `topRight`（safe 内），中间留给灵动岛/挖孔。
- 底栏 + 动作条：锚 `bottomCenter`，可点区在 `safeContentRect` 底边之上。
- 弹层（PawConfirm / 明信片查看器 / 商店卡）：`center` 锚定，宽度 = `min(safeContentRect.width - 2*gutter, maxCardWidth)`，超宽平板上不铺满、居中留白。
- 邮箱、食盆等场景内可点物：锚定在**背景百分比坐标**上（随背景缩放移动），但需实时校验其屏幕投影落在 safe-area 内；若某极端比例下投影出界，回退到就近的 safe-area 边缘锚点 `[待细化]` 回退表。

# 4. 响应式布局系统

## 4.1 布局基准

- **设计基准**：竖屏 `1290 × 2796` 物理像素（= 430×932 pt @3x），对应 spec-art-overview §2.1 参考画布。UI 以 **pt 逻辑单位**标注。
- **safe-area 内栅格**：横向 **4 列弹性栅格**（gutter = 16pt，margin = 屏宽相关，见下），纵向按锚点带（顶栏带 / 场景带 / 动作条带 / 底栏带）。
- **响应式 margin**：`sideMargin = clamp(16pt, 屏宽 * 0.045, 40pt)`——窄机 16pt，宽平板封顶 40pt，防止平板上元素被拉太开。

## 4.2 缩放策略（等比 vs 拉伸的适用范围）

| 元素类别 | 缩放方式 | 说明 |
|---|---|---|
| 背景/场景/宠物/访客/水彩插画 | **等比（uniform scale）** | 绝不拉伸，见 §3.2 cover |
| UI 手账元素（按钮/贴纸/图标/徽章） | **等比缩放** + 锚点重排 | 图形本体等比；靠锚点和间距吸收比例差，不拉伸单个元素 |
| 容器/面板/列表（相册网格、商店卡列） | **弹性布局（flex/wrap）** | 宽度自适应：窄屏少列、平板多列（网格列数按可用宽度断点切换） |
| 文本块 | **换行重排 + 字号档位** | 不缩放拉伸文字，靠换行与字号断点 |

**关键区分**：视觉资产=等比；结构容器=弹性重排。二者组合实现「填满不变形」。

## 4.3 响应式断点（可用宽度 dp）

| 断点 | 可用宽度 | 相册/商店网格列数 | 备注 |
|---|---|---|---|
| Compact | < 600dp | 2 列 | 手机竖屏主力 |
| Medium | 600–839dp | 3 列 | 折叠内屏、小平板竖屏 |
| Expanded | ≥ 840dp | 4 列 | 大平板、折叠内屏横向 |

> 主屏 YardScene 本身不分栏（永远单一院子场景 cover）；断点只影响**列表/网格类二级屏**（图鉴/相册/商店）。`[待验证]` 列数与断点。

## 4.4 字体与热区最小尺寸（触控可达性）

| 项 | 最小值 | 依据 |
|---|---|---|
| 正文字号 | ≥ 15pt（等效） | 手账正文可读下限 `[待验证]` |
| 次要/落款字号 | ≥ 12pt | 明信片落款等 |
| **最小可点热区** | **≥ 44×44 pt（iOS HIG）/ 48×48 dp（Android Material）** | 全平台取 **48×48 dp** 为统一下限 |
| 相邻热区间距 | ≥ 8dp | 防误触 |
| 动作条 `TapeButton` | ≥ 56dp 高 | 主要动作，放大易点 |
| 宠物点击热区 | 视觉外扩 ≥ 8dp 的不可见 padding | 保证小宠物幼崽期也易点 |

> 缩放后若某图标视觉 < 48dp，其**可点命中框仍须 ≥ 48dp**（透明外扩），视觉与命中解耦。

# 5. 横竖屏与折叠屏方向策略

## 5.1 方向策略：竖屏锁定（Portrait Lock）

- **建议：全局竖屏锁定**（`portraitUp`，允许 `portraitDown` 视产品定）。理由：
  1. 核心体验（院子、养宠、手账 UI）是**竖向单院子场景**，横屏会破坏构图并新增一整套横向背景资产（成本翻倍，违背 §6 交付节流）。
  2. 竖屏锁定天然规避「旋转时重排导致短暂黑边/闪烁」风险，服务红线 1。
  3. 契合 spec-ux §0.1「竖屏为主，横屏不支持」。
- **唯一例外**：明信片查看器（S8）允许**临时横持放大看插画**（sensor 横向仅此一屏，退出即恢复竖锁），对齐 spec-ux §1 S8「可横持放大看插画」。此横屏态同样 full-bleed 无黑边（横向背景=明信片插画本身按 cover）。

## 5.2 平板处理

> **iPad 新目标已升级**：通用设备红线仍以本文件为准，但 iPad 横竖屏、Split View、Stage Manager、宽屏多栏布局的产品级规格见 [`spec-ipad-adaptive.md`](spec-ipad-adaptive.md)。本节保留为旧的最低兜底策略；正式 iPad 支持不再把「平板竖屏锁定」作为目标。

- 平板**仍竖屏锁定**，不做横屏专属布局（避免双套设计）。
- 平板竖屏时比例偏方（16:10~4:3）：走 §3.2「过方」解法——背景按高度铺满、左右溢出裁切；UI 安全框变宽、`sideMargin` 封顶 40pt、网格列数升到 3–4。
- 不强制全屏拉伸 UI：弹层/卡片居中留白，避免元素被无意义拉宽。

## 5.3 折叠屏处理

| 场景 | 处理 |
|---|---|
| 外屏（窄长，类 F） | 按手机长屏正常处理；竖屏锁定 |
| 内屏展开（近方，类 G） | 按平板竖屏处理（§5.2）；背景 cover 吃掉横向溢出 |
| **折叠/展开切换（fold state change）** | 监听尺寸变化，触发**平滑 re-layout**：背景 `scale`/`offset` 重算并用 ≤300ms `WashTransition` 式淡入过渡，**过渡帧也不得露黑边**（旧帧保持 cover 直到新帧就绪）`[待细化]` |
| 铰链遮挡（Fold 内屏中缝） | 关键内容与宠物主体**避开屏幕水平中线 ± 24dp**（铰链/中缝区），避免主体被折痕切分 `[待验证]` |
| 状态保留 | 折叠/展开不丢失游戏状态（YardScene 后台态保留，见 spec-ux §S1） |

# 6. 与美术协作（回吐给美术规格的交付要求）

> 引用 `spec-art-overview.md` §2.1（全屏满幅、绝不留黑边、参考画布 1290×2796、@1x/@2x/@3x）。**本节不修改该文件**，只把设备适配对美术的**具体交付要求**明确化，供 spec-art-* 分域清单落实。

## 6.1 背景/场景类资产（full-bleed）交付要求

- **含溢出余量的「大画布」交付**：每张全屏背景以基准 `1290×2796` 为**核心安全构图区**，向四周额外扩绘：
  - 上下各 ≥ 12% 延展（天空/地面自然延续）。
  - 左右各 ≥ 15% 延展（远景/植被自然延续）。
  - → 建议实际交付画布 ≈ `1290×(1±0.12)` 宽 × `2796×(1±0.15)`... **规格：核心区 1290×2796 居中，外扩后总画布约 ≈ 1680×3350 `[待验证]`**（按上述比例）。
- **标注三条参考线**（随源文件 PSD 图层附）：
  1. `core-safe`：核心构图区（1290×2796），主体必须在内。
  2. `content-safe`：safe-area 内容区（供 UI 锚定参考，即扣除刘海/手势条后的区域）。
  3. `crop-bleed`：可裁切延展区边界（cover 到极端比例时会被裁掉的部分，**不得放不可裁的主体**）。
- **禁止**在延展区放置唯一关键信息（如唯一的邮箱/宠物），关键锚点物必须在 `core-safe` 内。
- 源文件按 @3x 起绘（对齐 §2.1），导出 @1x/@2x/@3x。

## 6.2 多比例适配交付

- 每张 full-bleed 背景需通过「4:3 ~ 21:9 裁切预览」自检：在最方与最长两个极端裁切框下，构图仍成立、无重要元素被裁、无空洞。附**两张极端比例的裁切预览图**供核对 `[待细化]`。
- 场景内可放置物（食盆/邮箱/装饰）交付**相对锚点百分比坐标**（相对 core-safe），而非绝对像素，供 §3.3 百分比锚定。

## 6.3 UI / 明信片资产

- UI 手账元素（按钮/图标/徽章）：等比缩放使用，交付时保证 @3x 下 ≥ 48dp 命中框对应的视觉清晰度；小图标附透明外扩说明（视觉可小于命中框）。
- 明信片插画（S8 横看态）：需同时满足竖排缩略与横持放大两种 cover 呈现，横向也不留黑边（插画本体做 cover）。

## 6.4 折叠屏/平板增量

- 无需为平板/折叠内屏重画横向场景（竖屏锁定，§5）；但 full-bleed 背景的**左右延展区余量**是折叠内屏/平板不黑边的唯一保障，§6.1 的 15% 左右余量为**硬性下限**，平板重度覆盖时建议留到 20% `[待验证]`。

> 本节仅占位分档与降级挂钩点，**具体阈值与开关清单见 `spec-performance.md`** `[待细化]`。分档不影响「无黑边/不变形/不遮挡」三红线——降级只降画质/动效，不改布局与填充。

# 8. 验收标准

## 8.1 跨设备可测验收清单（发布前逐条勾选）

**A. 无黑边（红线 1）**
- [ ] 在 §8.2 全部关键机型上，冷启动首帧 + 主屏 + 每个二级屏 + 每个演出层，**四条屏幕边缘均无黑边**（截图逐边检查，无 Scaffold 底色露出）。
- [ ] 折叠屏展开/折叠切换的**过渡帧**无黑边（录屏逐帧）。
- [ ] 明信片横持态无黑边。

**B. 无变形（红线 2）**
- [ ] 宠物在最方（4:3）与最长（21:9）比例下**长宽比一致**（同一宠物截图叠加比对，无胖瘦差异）。
- [ ] 水彩纹理/圆形元素（暖绒图标、徽章）在各比例下不呈椭圆/拉伸。

**C. 不遮挡 / 热区可点（红线 3 + 可达性）**
- [ ] 顶栏名牌/经验条/暖绒**不被灵动岛/刘海/挖孔覆盖**（灵动岛机型中央区无可点元素）。
- [ ] 底栏/动作条**不被手势条覆盖**，全部可点。
- [ ] 四角圆角处无关键文字/控件被切。
- [ ] 全部主要控件命中框 ≥ 48×48dp（自动化尺寸断言）。
- [ ] 曲面屏（类 C）边缘控件不落入曲面畸变区。
- [ ] 折叠内屏关键内容避开铰链中缝。

**D. 内容完整**
- [ ] 明信片正文、成长手账、商店卡文字在所有断点下完整可读、不截断、字号 ≥ 下限。
- [ ] 相册/商店网格列数随断点正确切换（2/3/4）。

**E. 方向**
- [ ] 全局竖屏锁定生效；仅明信片查看器允许横持，退出恢复竖锁，无残留横屏。
