# Petopia 实现规格 · 留存与长期动力 · v0.3 · 配套 DESIGN.md

> 本文件是给设计/开发者看的**留存层实现规格**：把「新手引导深化 + 温和留存 + 回归 + 长期动力/Endgame」落成可执行的脚本、节奏、文案与验收点。
> 规则/数值以 `DESIGN.md`（v0.3）为准，交互红线以 `spec-ux.md` §5–§7 为准；本文件不重复定义系统规则，只把它们编排成**玩家旅程**。
> 埋点与指标细节以 `spec-analytics.md`（`[待细化]` 待建）为权威来源，本文相关处均以 `[待细化]` 占位。
> 标记：`[待细化]` = 结构已定、细节待补；`[待验证]` = 数值/节奏需 playtest 校准。

## 0. 最高原则（留存哲学的硬约束）

> 本文所有机制的最终仲裁标准是 `DESIGN.md §0.3` 设计支柱。留存靠「**想念**」而非「**怕错过**」。

### 0.1 一句话

> 让玩家「**想**」打开，而不是「**该**」打开。回来永远是拆礼物，离开永远不受罚。

### 0.2 禁止清单（红线，任何机制不得触碰）

以下机制在 Petopia **明确拒绝并禁止**，评审时逐条对照否决：

- ❌ 体力条 / 心跳条 / 精力值等「消耗-恢复」资源门。
- ❌ 限时 FOMO 倒计时（「还剩 02:59:59 领取」）、限时活动逼你此刻上线。
- ❌ streak（连续登录）**断裂惩罚**——连登可作为「回忆里程碑」记录，但中断只是「最长连续段」不清零、不掉落、不扣任何东西（对齐 `ach_login_*` 口径）。
- ❌ 「每日不来就损失」——不掉心情、不变脏变饿、不掉好感度、不腐烂库存。
- ❌ 红点 / 数字角标轰炸 / 连续弹窗（对齐 `spec-ux.md §7.2`）。
- ❌ 开箱 / 抽卡 / 转盘等随机付费获取。
- ❌ 进度墙逼氪、「立即完成」加速付费（对齐 `DESIGN.md §4.4`）。
- ❌ 召回骚扰通知（「你 N 天没来 TA 很伤心/很想你」，对齐 `spec-ux.md §7.2/§6.2`）。

### 0.3 「正向替换」对照表（把常见留存套路换成零焦虑写法）

| 一线常见套路 | Petopia 的替换 | 差异本质 |
|---|---|---|
| 每日签到领奖（不来损失） | 登录「暖绒小心意」= 给予而非索取，攒着等你（§2.4） | 给予 vs 索取 |
| streak 火苗断了清零 | 连登只记「最长段」，中断无罚，回来照常（§0.2） | 记录 vs 惩罚 |
| 限时活动倒计时 | 季节回访/限定明信片：错过 = 明年再见，非「永久错过」（§4.4） | 循环 vs 一次性 |
| 体力耗尽劝氪 | 冷却仅水彩沙漏，离线只涨经验（`DESIGN.md §3.1/§3.2`） | 无上限门 |
| 红点催你清 | 邮箱旗子立起 + 柔光，不催不响（`spec-ux.md §0.2`） | 邀请 vs 催促 |
| 回归礼包「补偿你的损失」 | 积压明信片像拆礼物、宠物「自己玩了几天」（§3） | 惊喜 vs 补偿 |

## 1. FTUE 深化（首次会话逐分钟脚本）

> 承接 `spec-ux.md §5`（S15 Onboarding）。本节把「冷启动→领养→第一次照料」细化到**逐分钟脚本**、**aha 时刻锚点**、**首日/次日钩子**。
> 前提红线（不可破）：无账号、无登录、无网络权限弹窗、无强制箭头轰炸（`spec-ux.md §5.3`）。

### 1.1 首次会话逐分钟脚本（0–5 分钟）

| 时间 | 玩家做什么 | 看到 / 听到什么 | 设计意图 |
|---|---|---|---|
| **0:00–0:20** | 冷启动，不做任何操作 | 品牌水彩淡入 + 一句话「一个小院子，一次好好爱一只」；翻纸/铅笔沙沙音 | 建立手账水彩基调，无 loading 焦虑、无隐私弹窗 |
| **0:20–0:50** | 轻点翻过 2 张旁白手账卡 | 卡1「这里是你的小院子。慢慢来，没有输赢。」；卡2（情感锚点）「TA 长大后会去看世界——但那不是告别……你离开的时候，TA 只会睡个好觉，攒点暖绒等你。」 | **一次性种下三颗种子**：无输赢 / 离线有益 / 毕业非告别。非条款、非教程 |
| **0:50–1:40** | 从 3 个高亮初始物种（橘猫/柴犬/垂耳兔）里挑一只；其余占位格灰剪影/问号照常显示 | 图鉴贴纸册，3 格发光、其余格给「后面还有更多朋友」的期待感线索 | 决策轻（只有 3 选 1），同时用占位格埋长线钩子（不催促） |
| **1:40–2:20** | 看随机生成的个体（变体+2 性格贴纸+人设一句话）；可「换一个样子」1 次；取名 | 「TA 会记得这个名字一辈子」温柔提示 | **让玩家投入命名 = 情感所有权的第一次注入**。性格随机 = 每只独一无二 |
| **2:20–2:40** | 看入住演出（无操作） | 幼崽 `WashTransition` 落入院子，顶栏出现名牌（「幼」木牌、蛋壳纹徽章） | 「这是**我的**、有名字的、活着的小家伙」——所有权确立 |
| **2:40–3:20** | 跟随一次性引导气泡①：摸摸 TA | 点摸头 → 性格专属反应动画（如「眯起了眼」）+ `SoftToast +1` + 经验条微涨 | **★ 第一个 aha 时刻（触感反馈）**：我的动作 → TA 有反应、我在陪伴 |
| **3:20–4:10** | 引导气泡②：喂点东西 | 点食盆/喂食 → 走到碗边「吃得喷香」动画 + `SoftToast +3` + 手账新增一条 | aha 延续：照料有可见成长（经验条 + 成长手账），且**冷却以水彩沙漏温柔呈现**（第一次遇到「等一等」不焦虑） |
| **4:10–4:40** | 引导气泡③：指向邮箱 | 「邮箱以后会有远方的信。」邮箱静静立在院里（此刻无新片） | **埋下第二天/毕业后的钩子**：远方会来信，制造温柔期待 |
| **4:40–5:00** | 自由观察院子 | 宠物回到 IDLE 待机、播性格待机动作；也许一只常见访客（麻雀/蜗牛）正巧路过 | 交还操控权，让玩家发现「即使我不点，这个小世界也活着」 |

> **aha 时刻定位**：主 aha 在 **2:40–3:20 的第一次摸头反应**（情感触点，而非任何数值）。第二 aha 是 **3:20 的成长手账首条记录**（我的陪伴被认真记下来了）。二者都在 **首次会话 3.5 分钟内**出现，满足「≤2 分钟完成看开场→领养→第一次摸头」（`spec-ux.md §5.3`）后立即给情感回报。

### 1.2 情境化引导原则（非说教）

- **规则用「体验」揭示，不用「文字」宣讲**：冷却不解释机制，只给沙漏 + 「TA 还在回味呢～」气泡；离线收益第二天用「TA 睡饱了，蹭蹭你 +N」动画自解释。
- **气泡 ≤ 3 个、每个 ≤ 12 字、指向具体物件**（对齐 `spec-ux.md §5.2`）；用后永不重复。
- **首次会话不推商店/成就/图鉴长线说明**（首次毕业前不强推，`spec-ux.md §5.2`）——避免信息过载与「任务清单感」。
- **零箭头轰炸**：用柔和高亮光晕替代强制指引箭头。

### 1.3 首日目标（Day 0，全部可选、不做不罚）

> 呈现为「今日院子」软目标卡（§2.3），**不是任务列表**，无红点、无「未完成」警告。

| 软目标 | 触达方式 | 完成回报 |
|---|---|---|
| 认识你的新朋友（摸头 1 次） | 引导气泡① | 性格反应动画 + 首条手账 |
| 喂 TA 第一顿饭 | 引导气泡② | +3 经验、看「吃得喷香」 |
| 逛一圈院子（发现邮箱/食盆） | 引导气泡③ + 自由观察 | 认识长线入口，无强制 |
| （隐性）看见第一位访客 | 概率触发 | 来客图鉴收录 1 种（惊喜，非目标项） |

> Day 0 **不设「必须升级」目标**，避免把 2–3 周的慢节奏在第一天就压成 KPI。

### 1.4 次日回访钩子（Day 1 return hook）

> 目的：让玩家**想**回来，靠「TA 想你了 + 有新东西发生」，不靠「不来就亏」。

**次日首次上线时的黄金 20 秒（按序演出，不并发轰炸）：**

1. **离线收益暖场**：「TA 睡饱了，蹭蹭你 +N」动画（`DESIGN.md §3.2`，`spec-ux.md §7.3`）——回来先被给予。
2. **一条「离线小故事」**（§2.1）：宠物在你离开时发生的一件小事（「昨晚它把窝挪到了能看见你房间灯的位置」——复用 ev_d18 类文案，离线态改写为「想你」旁白）。
3. **最多 1 个当日 DAILY 事件**演出（`DESIGN.md §9.1`）。
4. 若前一日埋了「邮箱以后会有信」——**首张明信片不在 Day 1 强来**（宠物尚未毕业，无信可寄）；改由 Day 1 的**第一位访客高概率登场**承接「院子有新鲜事」预期。`[待验证]` Day 1 访客保底概率。

**次日通知（若开启，且严格 ≤1 条/日，可全关，`spec-ux.md §6`）：**
- 优先级：若 Day 1 无明信片/回访/特殊事件 → **不推任何 KPI 式召回通知**；仅当有真实内容（如特殊事件）才推。宁可不推，也不制造「快回来」压力。
- `[待细化]` 是否为「首周」提供一条极温和的、可一键永久关闭的「院子早安」通知（默认**关**，opt-in）——需 playtest 确认不越过召回红线。

### 1.5 D1 留存设计意图（目标值仅作设计参考，非后端指标）

> 以下数值是**设计自评参照**，不接入服务器、不做 KPI 考核；纯本地下的轻量自测见 §7。

| 参照维度 | 设计参照值 `[待验证]` | 达成靠什么（本文机制） |
|---|---|---|
| 首次会话完成领养+首次照料 | ≥ 90% | FTUE 决策轻、aha 早（§1.1） |
| D1 自然回访（含无通知） | 设计参照 ≥ 40% | 次日「想你了+离线收益+访客」钩子（§1.4） |
| 首周内首次「主动发现」访客/事件 | ≥ 70% | 院子「活着」的自运转（§1.1 4:40–5:00） |

> 意图声明：我们**不为了拉高 D1 而牺牲零焦虑**。若某机制能提升 D1 但依赖 FOMO/惩罚，一律否决（§0.2）。

## 2. 温和留存层（每天「想」打开的理由）

> 核心：把「每日义务」全部替换为「每日惊喜 / 每日给予 / 每日可选小目标」。所有项**离开多久回来都不亏**。

### 2.1 宠物离线小故事 /「想你了」

- **触发**：每次上线，若距上次在线 ≥ 一定时长（`[待验证]` 建议 ≥3h），在离线收益暖场之后，**低频**（非每次）弹一条离线小故事手账卡。
- **内容库**：复用 `content-events.md` 中「黏人/温柔」系事件文案（如 ev_d18/ev_d53/ev_d55/ev_d99），改写为**离线态旁白**（「在你不在的时候……」）。目标 `[待细化]` 独立离线小故事 ~20 条，按性格加权。
- **红线**：只讲「TA 自己过得很好 + 想你」，**绝不**讲「TA 很孤单/很伤心/等你等到……」。示例：
  - 黏人：「你不在的时候，它把你掉的纽扣叼进窝里守了两天。」
  - 高冷：「它才没有一直看门口呢。（它看了。）」
  - 慵懒：「这两天它主要在忙一件大事：把同一个午觉睡完。」
- **数据**：不产生经验/暖绒（避免变成「必须来领」的义务）；纯情感反馈。`[待验证]` 是否偶尔附赠极小暖绒（倾向**不加**，保持「给予是心意不是任务」）。

### 2.2 每日一件小惊喜（Surprise of the Day）

> 「每天打开都有点新鲜」，但**新鲜是被给予的，不是被要求领取的**。三选一按当日情境命中，最多 1 件/自然日：

| 小惊喜 | 来源 | 触达 |
|---|---|---|
| 一位访客路过 | `DESIGN.md §8.2` 到访判定 | 院子里 z5 层出现，可互动（`spec-ux.md §4.4`） |
| 一张明信片（有毕业宠时） | `DESIGN.md §6.1` 主旅程每 3–5 天，补完期每 10–15 天，长期约 20 天 | 邮箱旗立起（非红点），拆信像拆礼物 |
| 一个日常事件 | `DESIGN.md §9.1` 每日 1–3 个 | 手账卡从纸面浮现 |
| （偶发）回访宠串门 | `DESIGN.md §7` 每 7–14 天 | 铃铛声 + 老朋友身影 |

- **保底温柔感**：即使某天以上都没命中，宠物的**待机动作 + 离线暖场**本身已是「今天也好好活着」的反馈；**不制造「今天什么都没有」的空洞感**——ev_d100「今天什么也没发生……但你知道这就是最好的一天」正是这种调性的官方背书。
- **红线**：小惊喜**不可**做成「限时领取，过期作废」。明信片/访客错过当次上线，仍在邮箱/事件队列里等你（`spec-ux.md §6.2`）。

### 2.3 「今日院子」温和小目标（可选，不做不罚）

- **形态**：主屏一个可折叠的小手账便签（默认收起，点开才展开），列 1–3 条**当日可做的小事**，例：「摸摸 TA」「换一盆访客爱吃的食物」「去看看远方寄来的信」。
- **硬约束**：
  - **纯建议语气**，无「任务」「完成度 0/3」「未完成」字样；用勾选变成水彩小对勾即可，不做进度百分比。
  - **不做不产生任何损失**：不掉暖绒、不断连登、不弹提醒。
  - 完成给已有系统的自然回报（经验/`SoftToast`），**不额外设「日目标奖励」**（避免变成打卡义务）。
  - 无红点、无角标（`spec-ux.md §7.2`）。
- **意图**：给「不知道今天玩什么」的休闲玩家一个**温柔的起点**，同时对「想自己玩」的玩家完全隐形（默认收起）。

### 2.4 登录「暖绒小心意」（给予，而非索取）

> 替换业界「每日签到」。核心差异：**这是院子/宠物送你的心意，不是你必须来领的工资**。

- **机制**：每自然日**首次照料**（任意动作）触发 +5 暖绒（复用 `DESIGN.md §4.2` 已有「每日首次照料 +5」，本文只是明确其**留存语义**与文案）。
- **文案红线**：呈现为「TA 今天也给你掉了一小撮暖绒」的**给予叙事**，**禁止**「签到成功」「连续第 N 天」「今日已领」等打卡措辞。
- **不来无损失**：没上线的那天不产生暖绒，但**不倒扣、不断连、不提示错过**。攒着的暖绒永远在账上等你（`DESIGN.md §0.3` 只增不减）。
- **与连登成就的关系**：`ach_login_7/30/100` 记录「最长连续段」作为**回忆里程碑**，中断不清零（`content-achievements.md` 口径）——它是「纪念你陪了这么久」，不是「别断了否则……」。

### 2.5 每日 / 每周节奏设计

**每日节奏（一天 2–4 次、每次 2–5 分钟，`DESIGN.md §0.2`）：**

| 时段 | 玩家典型行为 | 系统供给 |
|---|---|---|
| 早（醒来） | 上线领离线暖场 + 首次照料（暖绒小心意）+ 看晨间事件 | 晨间权重事件（ev_d13/d48/d79…）、访客晨窗（6–9 点判定） |
| 午 / 碎片时间 | 冷却已回、补一轮照料、看有无明信片 | 午间事件、明信片到达 |
| 晚 | 夜间事件、夜间访客窗（18–21 点）、回访常在夜里更有氛围 | 夜间权重事件（ev_d07/d18/d61…）、夜行访客、特殊事件 |

- **冷却编排**：喂食 15min / 摸头 10min / 玩具 20min 的错位冷却（`DESIGN.md §3.1`）天然形成「回来一趟能做几件事、做完自然告一段落」的节奏——**主动促成「自然停手」而非「无限刷」**，这是零焦虑节奏的一部分。

**每周节奏：**

| 周内节点 | 内容 | 来源 |
|---|---|---|
| 贯穿全周 | 经验缓慢累积、离线收益、日常事件 | `DESIGN.md §3.3` 曲线 |
| 主旅程每 3–5 天 | 一张明信片（有毕业宠后） | `DESIGN.md §6.1` |
| 每 7–14 天 | 一次回访串门 | `DESIGN.md §7` |
| 约第 2 周 | 换模演出（Lv5/8）、成年礼等成长节点 | `DESIGN.md §3.6`、ev_s15/s16 |
| 约每 7 天 | 生日会（ev_s02，领养满 7 天）等小仪式 | `content-events.md` |

> 设计意图：每周至少有 **2–3 个「非我操作驱动」的温柔事件**（明信片/访客/回访/特殊事件）主动来找玩家，让「打开」总有新鲜期待，而这些期待**不会因为晚来几天而消失**。

## 3. 回归机制（Win-back，离开数天后回来）

> 场景：玩家离开 3 天 / 7 天 / 30 天后回来。原则：**回来是拆礼物，不是收账单**。绝无「补偿你的损失」框架（因为根本没有损失）。

### 3.1 回归时的黄金演出序列（离开 ≥3 天，按序、不并发）

1. **宠物没事，只是自己玩了几天**：上线先播「TA 睡饱了/自己玩了几天，蹭蹭你 +N」离线暖场（离线经验按 `DESIGN.md §3.2` 结算，最后一段封顶 + 每日上限——**离开 3 天和 3 小时收益上限一致，这本身就是零焦虑的证明**，但演出上要**强调 TA 过得很好**）。
2. **积压明信片像拆礼物**：若离开期间生成了多张明信片，邮箱旗子立起（非红点），进入后**逐张拆信演出**（`spec-ux.md §4.3`），像「一次收到一叠远方来信」的惊喜，而非「N 条未读待清理」。`[待验证]` 积压 >5 张时提供「一次铺开看」的相册化收信，避免逐张演出过长。
3. **一条「这几天 TA 的小故事」**（§2.1 离线小故事的加长版）：把「自己玩了几天」讲成温暖片段（「这几天它学会了自己把窝挪到有太阳的地方」）。
4. **可能积压的回访 / 特殊事件**：按 `spec-ux.md §2.3` 演出排序（换模 > 毕业 > 事件 > 回访）逐个补演，**不一次性轰炸**。
5. **不出现任何**：「你已 N 天没来」「TA 好想你/好难过」「损失了 X」——**禁止**（§0.2，`spec-ux.md §7.3`）。

### 3.2 回归文案调性（按离开时长，全部温暖、零指责）

| 离开时长 | 暖场文案示例 | 禁止写法 |
|---|---|---|
| 3–6 天 | 「TA 这几天把你的院子看得好好的，就等你回来。」 | ❌「你 3 天没来」 |
| 7–13 天 | 「远方寄来的信在邮箱里叠成了一小摞，TA 也攒了好多话想跟你说。」 | ❌「快回来领取过期奖励」 |
| ≥14 天 | 「院子还是老样子，TA 一眼就认出了你。（当然，它假装只是路过。）」 | ❌「TA 差点忘了你 / 很伤心」 |

> 说明：回归**不重置**任何进度、**不损失**任何东西、**不加速追赶**（无「离线补偿宝箱」逼你立刻消费）。回来即无缝续玩。

### 3.3 回归通知（win-back notification，严守通知红线）

- 对齐 `spec-ux.md §6`：**每日至多 1 条、可全部关闭、无召回骚扰**。
- **明确禁止**任何「N 天没来」触发的召回推送（`spec-ux.md §7.2`：召回类通知是**禁止项**）。
- 回归只能靠**真实内容通知**自然带回：即玩家离开期间若产生了明信片/回访/特殊事件，仍按「每日 ≤1 条、内容驱动」推送（如「邮箱里有新明信片，是从远方寄来的。」）——**推的是「有信来了」这件好事，不是「你该回来了」这个指责**。
- `[待细化]` 是否对「长期未回」玩家做一次**极温和、可永久关闭**的单次问候（如「院子的花开了。」不含宠物想你、不含 KPI）——默认保守，需 playtest 与红线复核，倾向**不做或默认关**。

## 4. 长期动力 / Endgame（养满 12 种、集齐之后玩什么）

> 场景：玩家已养满 12 种（含彩蛋）、豪华度⑥「传说中的院子」、图鉴/明信片/邮戳趋于集齐。
> 调性总纲：**没有硬核终局也不焦虑**——Petopia 的「终局」不是打败什么，而是「一座住满回忆的院子」。完成度是**回忆的丰度**，不是待办的清空。

### 4.1 元进程（Meta-progression）总览

| 层 | 驱动 | 完成后仍持续？ | 表达 |
|---|---|---|---|
| 宠物毕业代数 | 一只只养 | ✅ 无上限，可无限领养（含重复物种） | 图鉴历任名字列表持续加长、旅行相册持续增册 |
| 院子豪华度 | 累计毕业数（`DESIGN.md §5.1`） | ⑥ 为当前顶 → 见 §4.2 延伸 | 纪念墙 + 全景 |
| 收集完成度 | 图鉴/访客/明信片/邮戳/成就 | ✅ 见 §4.3 完成度体系 | 手账装帧升级、里程碑贴纸 |
| 季节循环 | 真实日期弱同步 | ✅ 永远循环 | 季节限定明信片/事件/访客回访（§4.4） |

### 4.2 豪华度⑥之后：「纪念花园」延伸（Post-⑥）

> `DESIGN.md §5.1` 豪华度⑥「传说中的院子」是当前顶，含纪念墙。本节设计**⑥之后仍有温柔进展**，避免「满级即空」。

- **纪念墙持续生长**：每再送一只毕业，纪念墙新增一幅水彩肖像 + 该宠一句「毕业赠言」（从明信片文风生成）。墙可无限延展成「回忆长廊」，`[待细化]` 分屏/卷轴滚动查看。
- **纪念花园（Memorial Garden）**：⑥ 后解锁，**每毕业一只宠物，院子里长出一株属于它的植物**（花色/形态由物种+性格决定，如贪吃橘猫→一株结「叉烧色」果子的小树的水彩戏谑表达 `[待细化]`）。花园是「你爱过的所有生命」的活体图鉴，纯观赏、可点每株看是哪只宠物。**无维护、不会枯萎**（零焦虑：种下就永远在）。
- **历任宠物「同学会」（Reunion Day）**：`[待细化]` 低频特殊事件——多只毕业宠在同一天/同一周先后回访时，触发一场「老同学聚会」演出（呼应隐藏成就 `ach_h_classmates`「同学会」）。数只毕业宠同框在院里各玩各的，玩家旁观。纯情感、无收集压力。
- **`[待细化]` 豪华度⑦+**：是否在⑥之上再加「灯火长明的院子」等纯美术里程碑档（无功能、只是院子更繁茂），供超长线玩家继续看到「陪伴的累积在变美」。倾向做成**纯视觉递进**，不解锁玩法门槛。

### 4.3 收集完成度体系（Completion）

> 把「集齐」做成**里程碑纪念**，而非「缺口警告」（`spec-ux.md §7.6`：未解锁用期待感线索，不用红叹号）。

| 完成度维度 | 里程碑 | 纪念表达（非奖励，是回忆） |
|---|---|---|
| 宠物图鉴 | 8 常规 / 12 全 | `ach_unlock_all_regular` / `ach_grad_12`；图鉴金封面、纪念墙金边 |
| 单物种深度 | 同物种 3 只 / 集齐 5 变体 | `ach_species_repeat_3`（家谱贴纸）/ `ach_variant_5`（烫金页） |
| 访客图鉴 | 20 全收 | `ach_visitor_all`（金封面） |
| 明信片 | 30 / 100 张 | `ach_postcard_30/100`（相册皮肤） |
| 邮戳（地点） | 20 / ~40 全到访 | `ach_stamp_20/40`（「环球邮差」皮肤） |
| 特殊事件 | 全 20 种 | `ach_special_20`（鎏金腰封） |

- **「完成度总览」手账页** `[待细化]`：一页水彩仪表盘，用**「已收集的回忆」正向计数**（「你已收藏 87 张明信片、送走 9 位朋友」），**不显示「还差 X 未完成」**的缺口式文案。
- **100% 完成后**：解锁一枚「**满院花光**」纪念徽章 + 一段结语手账（`[待细化]` 文案：大意为「院子已经住满了故事，但故事永远可以再多一个」）——**明确不弹「游戏结束」**，玩家可继续无限养育、季节循环、纯享受院子。

### 4.4 季节限定回访（循环而非一次性）

> 关键：季节内容是**循环**的——错过今年，明年同季再见。**不是限时 FOMO**（对齐 §0.3 循环 vs 一次性）。

- 春/夏/秋/冬各有专属特殊事件（ev_s11/s12/s13/s08·s14）、季节限定访客（萤火虫夏、雪兔冬、白粉蝶春夏）与季节明信片滤镜。
- **限定不作废**：季节访客/事件是「今年这个季节的老朋友又来了」的温暖复现，**下季/明年会再来**，绝不写「本次限定，永久错过」。
- 长线玩家的动力：集齐「四季来信」`ach_season_4`、见证同一只毕业宠**跨季节**寄回不同季节风景的明信片。

### 4.5 传承 / 新周期的温柔表达（Legacy）

- **无「重开/新周期清档」**：Petopia 不做 rogue-like 式重置。所谓「新周期」= **领养下一只**，一切旧回忆（图鉴/相册/纪念墙/花园）**永久保留并叠加**。
- **传承叙事**：新幼崽入住时，`[待细化]` 低频可触发「上一届毕业宠回访，把某件旧玩具/习惯传给新崽」的温柔演出（呼应 ev_s16「成年礼」交还旧玩具的调性、ev_s07「老朋友的信物」）。
- **「一次只爱一只」的长期回报**（`DESIGN.md §0.3` 支柱③）：单宠位确保每只被记住；长期玩家收获的是**一整墙被认真记住的名字**，这本身就是最强的长期动力，无需硬核终局。

### 4.6 Endgame 调性声明

> Petopia 承认并拥抱「玩家总有一天会玩得比较少」。我们的设计目标**不是**把玩家永远锁在每日循环里，而是让**每一次回来**（哪怕一个月一次）都温暖、无门槛、有新明信片可拆。**「可以随时离开、随时回来、都不亏」本身就是我们要长期兑现的承诺**——这比任何 endgame 数值都更能让玩家在数月后仍愿意想起这座院子。

## 5. 玩家类型 × 动机满足映射

> 三类典型玩家「被什么留住」。所有满足项都严格落在零焦虑框架内。

| 玩家类型 | 核心动机 | 被什么留住（本文/已有机制） | 要避免对 TA 做的事 |
|---|---|---|---|
| **休闲 / 陪伴型** | 轻松的情感陪伴、低压力 | 每天 2–5 分钟的照料手感、离线无损、「今日院子」软目标（可无视）、离线小故事「想你了」、随时回来都无缝（§1、§2.3、§3） | 别塞任务清单、别催、别红点、别让 TA 觉得「落下了」 |
| **收集控** | 集齐、100% 完成度、稀有解锁 | 图鉴 4 态占位期待、彩蛋宠谜语线索、邮戳/变体/访客全收、完成度里程碑与装帧升级、纪念花园（§4.3） | 别把缺口做成「未完成警告」，别用抽卡随机化收集，别限时逼氪 |
| **剧情 / 情感党** | 故事、文案、离别与重逢的情感弧 | 100+ 事件手账文案、明信片性格文风叙事、毕业「离别是旅程」、回访重逢、离线小故事、同学会/传承演出（§4.2/§4.5） | 别把情感场景做成可跳过的 KPI，别在离别处写「失去」，别用情感勒索（想你=快回来）做召回 |

> 交叉说明：三类玩家共享同一套内容，靠**不同的呈现权重**满足——收集控看图鉴/完成度页，剧情党看相册/手账，休闲党只看院子主屏。没有任何一类被机制惩罚。

## 6. 参与度节奏曲线（单只 × 多只叠加）

### 6.1 单只宠物 2–3 周情绪/内容节奏

> 基于 `DESIGN.md §3.3` 经验曲线（Lv1→10 约 17–18 天）叠加事件/明信片节点。

```
情绪强度
 高 │      ●新鲜期          ●换模B    ●换模C    ★毕业典礼
    │     ╱ ╲             ╱ ╲       ╱          ╱ ╲
 中 │    ╱   ╲___平稳陪伴_╱   ╲__ __╱  ╲__温情_╱   ╲__旅行期开始
    │   ╱                              (回落风险区)      ╲收明信片
 低 │  ╱                                                  ╲___
    └──┼────┼────┼────┼────┼────┼────┼────┼────┼────┼──→ 天
       D0   D2   D4   D6   D8  D10  D12  D14  D16  D18  D20+
       领养  生日会前 Lv5    平稳   Lv8   毕业预演  毕业   旅行/回访
```

| 阶段 | 天数 | 情绪 | 内容供给 | **流失风险点** | 温和补救 |
|---|---|---|---|---|---|
| 新鲜期 | D0–D3 | 高（新奇、命名投入） | FTUE、前 3 级一天一级强正反馈（`DESIGN.md §3.3`）、首访客、生日会将近 | 低 | — |
| 第一平稳期 | D4–D7 | 中，可能开始平淡 | 日常事件、访客、生日会（ev_s02，D7）、Lv5 换模临近 | ⚠️ **首个回落**：新奇褪去、升级变慢 | 用 **Lv5 换模演出 + ev_s15「长大了一点点」** 作为 D6 前后的情感高点；生日会 D7 补一针温情 |
| 成长中段 | D8–D13 | 中，靠内容维持 | 事件密度、Lv8 换模 + ev_s16「成年礼」、可能首次回访（若已有毕业宠） | ⚠️ **最大回落区**：距毕业尚远、距上个高点已过 | Lv8 换模是锚点；**若玩家已有毕业宠**，此期的明信片/回访主动来「续新鲜」；「今日院子」软目标兜底 |
| 冲刺期 | D14–D17 | 升高（毕业在望） | ev_s20「同行半月」（D14）、ev_s06 毕业预演（Lv9）、旅装换模 | 低 | 毕业预演制造「即将启程」的温柔期待 |
| 毕业与旅行开启 | D18–D20+ | ★峰值（毕业）→ 转为期待（等信） | 毕业典礼、暖绒结算、旅行相册建册、毕业后 1 天首张明信片 | ⚠️ **毕业后空窗**：老宠走了、新宠还没养起来的情感真空 | **立即可领养下一只** + 首张明信片 1 天内到 + 豪华度进化让「新院子」有新鲜感 |

### 6.2 多只叠加（重叠养育 × 旅行/回访）——填平单只的谷

> 关键洞察：单只曲线的**谷**（D8–D13、毕业后空窗）正好被**上一只/下一只的旅行期内容**填平。这是「一次只养一只」却不无聊的核心设计。

```
        养第1只 ████████████▓▓▓▓ 毕业→旅行寄信············回访····回访····
        养第2只      （空窗被第1只的明信片填补）████████████▓▓▓▓ 毕业→旅行·····
        养第3只                                 ████████████▓▓▓▓ 毕业→·····
时间 →  ══╪═══════════╪═══════════╪═══════════╪═══════════╪═══════════
        ██ 养育期(高互动)   ▓▓ 冲刺/毕业   ···· 旅行明信片+回访(被动惊喜流)
```

- **叠加效应**：养到第 2 只时，第 1 只正在旅行期持续寄明信片（主旅程每 3–5 天，补完期每 10–15 天，长期约 20 天）+ 每 7–14 天回访——**新宠养育的平稳中段，恰好有老宠的明信片/回访不断带来惊喜**。玩家越玩越久，被动内容流越丰厚。
- **毕业空窗被消除**：第 1 只毕业后的情感真空，被「立刻领养第 2 只 + 第 1 只开始寄信」双重填补。
- **回访调度**（`DESIGN.md §7`）：多只毕业宠错峰回访（每只独立 7–14 天窗口，同时最多 1 只），使中后期玩家几乎每周都有「老朋友回来」。

### 6.3 流失点总表与补救原则

| 流失点 | 出现时机 | 补救（均为温和、非胁迫） |
|---|---|---|
| 首会话未建立情感 | D0 | FTUE 早 aha（§1.1）、命名投入 |
| 次日不回 | D1 | 「想你了」+离线收益+访客钩子（§1.4）；**不靠召回通知** |
| 首平稳期倦怠 | D4–D7 | Lv5 换模 + 生日会情感高点 |
| 中段最大回落 | D8–D13 | 若有毕业宠→明信片/回访续新鲜；软目标兜底；Lv8 换模 |
| 毕业后空窗 | D18–D20 | 立即可领养 + 首张明信片 + 豪华度进化 |
| 长期集齐后 | 数月 | 纪念花园/季节循环/同学会（§4）；**承诺「随时回来都不亏」** |

> 补救总原则：**只加「来了会更好」的正向内容，绝不加「不来会更糟」的负向压力**。

## 7. 留存指标框架（设计自评参考，非后端 KPI）

> Petopia 纯本地、无后端（`DESIGN.md §0.2`）。以下指标**仅作设计自评与 playtest 参考**，不接入服务器、不做增长考核；埋点细节以 `spec-analytics.md` 为准 `[待细化]`。

### 7.1 关心的指标（及零焦虑口径下的意义）

| 指标 | 定义 | 设计参照值 `[待验证]` | 在 Petopia 的意义 |
|---|---|---|---|
| D1 留存 | 领养次日回访率 | ~40% | 「想你了」钩子是否有效（§1.4） |
| D7 留存 | 第 7 天回访率 | ~25% | 是否熬过首平稳期到 Lv5/生日会 |
| D30 留存 | 第 30 天回访率 | ~12% | 是否进入「多只叠加」的被动内容流 |
| 平均会话时长 | 单次在线分钟 | 2–5 min | 是否符合「轻互动」定位（`DESIGN.md §0.2`）——**过长反而要警惕是否变成刷** |
| 日均会话次数 | 每日上线次数 | 2–4 次 | 冷却错峰是否形成健康节奏 |
| 回归率（win-back） | 离开 ≥7 天后回来占比 | `[待细化]` | 「随时回来都不亏」承诺是否兑现 |
| 首次毕业到达率 | 养到 Lv10 的玩家占比 | `[待验证]` | 单只完整弧是否被走完 |

> **口径警示**：本框架**刻意不追求「日活最大化」**。若某指标（如会话时长）异常偏高，需反查是否引入了违反零焦虑的「刷」诱因，而非庆祝。指标服务于「体验是否健康」，不服务于「留存最大化」。

### 7.2 纯本地下的轻量自测方法

- **本地事件计数器**：在设备本地记录关键行为计数（领养/毕业/照料/事件/明信片阅读/上线时间戳段），**仅存本地存档**，供开发调试面板与 playtest 回收，不上传。
- **playtest 日记法**：招募 `[待验证]` N 名玩家，用「本地导出存档 + 主观访谈」评估情绪曲线（§6）实际谷点与 FTUE aha 是否命中。
- **调试面板**（`[待细化]`，仅 debug build）：可视化本地 `ExpLogEntry` / 上线时间线 / 事件命中，用于自测节奏而非监控用户。
- **隐私红线**：任何计数仅本地、可随存档一并被玩家删除，**不做任何网络回传**（呼应 `spec-ux.md §5.3` 无网络请求）。

### 7.3 埋点占位

- 事件命名规范、字段、本地表结构、调试面板 UI：**`[待细化]`，与 `spec-analytics.md` 对齐**（该文档待建）。

## 8. 验收标准 + MVP 子集

### 8.1 验收标准（发布前逐条对照，与 `spec-ux.md §7.7` 红线联合勾选）

**A. FTUE 深化**
- [ ] 首次会话 ≤3.5 分钟内出现「第一次摸头反应」aha（§1.1）。
- [ ] 首日全程无强制账号/登录/网络请求（`spec-ux.md §5.3`）。
- [ ] 引导气泡 ≤3 个、用后不再出现；核心三点（离线收益/无惩罚/毕业非告别）以体验方式传达、无条款说教。
- [ ] 「首日目标」全部可选，无红点、无「未完成」文案、不做不罚。

**B. 温和留存层**
- [ ] 离线小故事只讲「TA 过得好+想你」，无「孤单/伤心/等你」写法（§2.1）。
- [ ] 每日小惊喜错过当次上线仍在邮箱/队列中等待，无「限时作废」（§2.2）。
- [ ] 「今日院子」默认收起，语气为建议非任务，无进度百分比、无红点（§2.3）。
- [ ] 登录暖绒为「给予」叙事，无「签到/连续第N天/已领」措辞；不来无倒扣（§2.4）。

**C. 回归机制**
- [ ] 离开 3/7/30 天回来：先播「TA 过得好+离线收益」，明信片以「拆礼物」呈现（§3.1）。
- [ ] 全程无「N天没来/想你/损失/补偿」文案（§3.2，红线 §0.2）。
- [ ] 回归通知仅内容驱动、每日≤1、可全关，无召回骚扰推送（§3.3，`spec-ux.md §6`）。

**D. 长期动力 / Endgame**
- [ ] 豪华度⑥后仍有温柔进展（纪念墙生长/纪念花园）（§4.2）。
- [ ] 完成度用「已收藏回忆」正向计数，无「还差X未完成」缺口式文案（§4.3）。
- [ ] 100% 完成不弹「游戏结束」，可继续无限养育与季节循环（§4.3/§4.6）。
- [ ] 季节限定为「循环复现」，无「永久错过」写法（§4.4）。

**E. 全局零焦虑复核**
- [ ] §0.2 禁止清单**逐条**未被触碰（体力条/FOMO倒计时/streak惩罚/每日损失/红点/抽卡/进度墙氪/召回骚扰）。
- [ ] 指标框架仅本地自测、无网络回传、不追求日活最大化（§7）。

### 8.2 MVP 子集（进入 MVP 的留存内容）

> 对齐 `DESIGN.md §12` MVP 范围精神：FTUE 与基础留存钩子进 MVP，长期/Endgame 与高级项延后。

**✅ 进 MVP：**
