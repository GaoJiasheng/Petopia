# 1. 全局声音美学与质量红线

## 1.1 总基调

- **一句话方向**：一本会发出声音的手账——温暖、松弛、有呼吸感的原声（acoustic）小品世界。参照《旅行青蛙》《猫咪后院》《Alba》《A Short Hike》的亲密感，以及《Sky 光·遇》级的制作完成度。
- **乐器家族（全项目统一音色库）**：毛毡钢琴（felt piano）、尼龙弦木吉他、卡林巴 / 八音盒、马林巴 / 颤音琴、竖笛 / 陶笛 / 单簧管（弱奏）、拨奏弦乐、手风琴（点缀）、口哨与轻哼唱、刷子轻打击（brushes / shaker / 手拍）。**禁止**：合成器主奏、电吉他、EDM 鼓组、8-bit 芯片音（彩蛋演出中的「星光系」音色允许少量 celesta / 玻璃质感 pad）。
- **和声与节奏**：大调 / Lydian / 五声为主；BPM 60–92；允许轻微 rubato 与「一顿一顿的笨拙感」演奏瑕疵（对齐美术的手绘出线 1–2px 精神）——但录音与母带质量必须专业级。
- **空间**：小房间 / 院子尺度的自然混响（RT60 ≈ 0.8–1.4s），亲密近距离拾音质感；禁止大教堂式史诗混响。

## 1.2 质量红线（对齐 spec-art §1.2，最高优先级硬性约束）

> **凡冲突，一律以本节为准。**

- **成品级交付**：所有音频必须是**可直接上架 App Store / Google Play 的终稿**——专业录音/制作、干净母带、无底噪爆音、循环无缝。禁止廉价罐头音效库直出、禁止 MIDI 直渲无混音、禁止占位音。
- **零焦虑声学红线（呼应 DESIGN §0.3 / spec-ux §7）**：
  - 禁止刺耳高频（任何资产 4–8kHz 不得有尖峰共振）、禁止惊吓式突强（单条资产内瞬时响度跃升 ≤ 12 LU）、禁止催促/警报/倒计时类音型（急促重复短音、上行警笛音、蜂鸣）。
  - 禁止「错误/失败」负反馈音——冷却中、余额不足、上限已达等状态**只用中性柔和音或无音**（见 §6）。
  - 通知音每日至多响 1 次，且必须可全关（spec-ux §6）。
- **一致性兜底**：全项目由 §1.1 音色库与 §2 技术规范统一；任何资产不得低于本质量线。交付前质检：响度达标、真峰达标、循环点无咔哒、无直流偏移、头尾无静音冗余（SFX 首 transient 距文件头 ≤ 10ms）。

# 2. 技术规范

## 2.1 格式与交付

| 资产类型 | 母带交付 | 打包运行时格式 | 说明 |
|---|---|---|---|
| 音乐（bgm_ / sting_） | WAV 48kHz / 24bit | **OGG Vorbis q6（≈192kbps）**；iOS 可选 AAC 256kbps | 分层曲目按 stems 交付（§2.4） |
| 环境音床（amb_） | WAV 48kHz / 24bit | OGG Vorbis q5 | 全部无缝循环 |
| 音效 / UI / 拟声（sfx_ / ui_ / voc_） | WAV 48kHz / 24bit | **WAV 16bit**（≤1s 短音，低延迟）或 OGG q6（>1s） | 变体打包为同名 `_01/_02/_03` |
| Stinger（sting_） | WAV 48kHz / 24bit | OGG Vorbis q6 | 单次播放，不循环 |
| Haptic（haptic_） | iOS：AHAP（Core Haptics）；Android：`VibrationEffect` 波形 JSON | 同左 | 与音效同名配对（§9） |

- 声道：BGM / amb 立体声；sfx / ui / voc **单声道**（由引擎按场景位置声像化）；stinger 立体声。
- 采样率全链路 48kHz；禁止上采样假高清。母带同时附 **循环元数据**（见 §2.3）与 stems 工程说明。
- 目录：`assets/audio/bgm/`、`/amb/`、`/sfx/`、`/ui/`、`/sting/`、`/voc/`、`/haptic/`，文件名 = `asset_id`。

## 2.2 响度标准（EBU R128 / ITU-R BS.1770-4 测量）

| 类别 | 积分响度目标 | 容差 | 真峰上限 | 备注 |
|---|---|---|---|---|
| bgm_ | **-16 LUFS** | ±1 LU | **-1.0 dBTP** | 动态范围 LRA 4–8 LU，保留呼吸感，不做砖墙压限 |
| amb_ | **-21 LUFS** | ±1.5 LU | -3.0 dBTP | 常年垫底，混音时再按场景推子（§10） |
| sfx_ | **-14 LUFS**（短时/瞬时按 Max-S 计） | ±1 LU | -1.5 dBTP | 交互反馈略高于音乐层 |
| ui_ | **-16 LUFS**（Max-S） | ±1 LU | -3.0 dBTP | UI 永远比交互 SFX 谦逊 2 LU |
| sting_ | **-14 LUFS** | ±1 LU | -1.0 dBTP | 演出高光，但瞬时不超引擎主限制器 |
| voc_ | **-16 LUFS**（Max-S） | ±1.5 LU | -2.0 dBTP | 拟声可爱优先，禁止刺耳峰 |

- 引擎主输出挂安全限制器 -0.5 dBTP；任何组合叠加不得触发可闻泵浦。
- 全类别高通 ≥ 40Hz（手机扬声器无效频段不留能量）；voc / ui 高通 ≥ 80Hz。

## 2.3 循环点标注（硬性要求）

- 所有循环类资产（bgm 全部 stems、amb 全部、sfx 中标注 `loop` 者）交付时**内嵌 WAV smpl chunk 循环点**，并在交付清单附 `loopStart / loopEnd`（样本数）。
- 循环必须**过零平滑**或附 ≥ 50ms 等功率交叉渐变尾巴；QA 标准：单曲连续循环 30 分钟无可闻接缝、无相位咔哒。
- 音乐循环点必须落在**小节边界**，并在元数据标注 BPM / 拍号 / 调性，供 §10 的对拍过渡使用。

## 2.4 自适应 BGM 分层 stems 交付约定

- 分层曲目按 **stems 分轨交付**：每轨独立 WAV，同一曲目所有 stems **等长、同 BPM、同起点、循环点一致**，任意子集叠加相位干净。
- stem 命名：`<曲目id>__st<序号>_<内容>`，如 `bgm_yard_day__st2_melody`。
- 每曲附一页「分层驱动表」：哪个 stem 由哪个游戏状态开/关（本文 §3 已给出，交付时随工程复述确认）。

## 2.5 命名规范（snake_case）

格式：`<域>_<主体>_<变体/状态>`

- 音乐：`bgm_yard_day`、`bgm_graduation`
- 环境：`amb_rain_soft`、`amb_insects_night`
- 音效：`sfx_feed_eat`、`sfx_stamp_press`
- UI：`ui_tape_button`、`ui_page_turn`
- 演出：`sting_evolve_d`、`sting_first_snow`
- 拟声：`voc_cat_happy`、`voc_visitor_owl`
- 触感：`haptic_pat`、`haptic_levelup`
- 变体后缀 `_01/_02/_03`；stems 后缀见 §2.4。

## 2.6 资产条目字段模板（本文各清单统一使用）

| 字段 | 含义 |
|---|---|
| `asset_id` | 命名规范下的唯一 ID |
| 用途 | 用在哪个系统 / 屏幕 / 时刻 |
| 风格与情绪描述 | 声音画面、乐器、情绪、关键细节（喂给音频模型/作曲人的 prompt 基础） |
| 技术参数 | 类别默认继承 §2.1/§2.2，条目内只标注例外 |
| 时长 / 循环 | 秒数或分秒；`loop` = 无缝循环，`one-shot` = 单次 |
| 分层 stems / 变体 | stems 数与内容；或随机变体数 |
| 关联 | 对应 DESIGN / spec-ux 章节 |

# 3. 自适应 BGM

## 3.1 院子主题曲套件（核心，自适应分层）

> **设计**：院子主屏（spec-ux S1）音乐 = 「一首会跟着院子长大的曲子」。按**时段**分 3 首姊妹曲（同主题动机、同调性家族 C 大调/A 小调、BPM 72），按**季节**换装饰 stem，按**院子豪华度**逐档点亮加花 stem，按**天气**做混音修正（§10.3）。玩家养到「传说中的院子」时，听到的仍是第一天那段旋律——只是变得丰盛了。

**通用 stems 结构（每首时段变体 6 轨）**：

| stem | 内容 | 驱动条件 |
|---|---|---|
| `__st1_bed` | 和声床：毛毡钢琴分解 + 尼龙吉他指弹 | 常开 |
| `__st2_melody` | 主旋律：卡林巴/八音盒（日）→ 陶笛（暮）→ 玻璃颤音琴（夜） | 常开；互动动画锁 / 事件卡演出期间 -6dB（§10.2） |
| `__st3_texture` | 空气感 pad：弱奏单簧管长音 + 纸张摩擦白噪的极轻织体 | 常开 |
| `__st4_perc` | 刷子轻打击 + shaker + 指尖敲木盒 | 豪华度②起开启 |
| `__st5_lux_a` | 加花 A：拨奏弦乐对位、口哨应答句 | 豪华度④起开启 |
| `__st6_lux_b` | 加花 B：手风琴长音 + 轻哼唱衬底（无歌词，哼鸣） | 豪华度⑥「传说中的院子」开启 |

**曲目清单**：

| asset_id | 用途 | 风格与情绪描述 | 时长/循环 | stems | 关联 |
|---|---|---|---|---|---|
| `bgm_yard_day` ★MVP | 院子主屏·白天（6:00–17:00） | 主主题。晨光晒在草皮上的松弛感：毛毡钢琴与吉他一问一答，卡林巴唱主歌，像有人在院子里边哼歌边晾衣服。中段留 8 小节「只剩床与织体」的呼吸段落 | 2:40 loop（小节边界） | 6 | DESIGN §5、spec-ux §3 |

# 8. 宠物拟声与访客拟声（Voc）

## 8.1 设计原则

- **真实动物声的「水彩化」**：以真实叫声为素材基底，做圆润化处理（去尖峰、加软化包络、微升可爱共振峰），保留物种辨识度；奇幻宠使用乐器/人声合成的「非语言小嗓音」。
- **性格化表达（对齐 DESIGN §2.3）不靠翻倍录音，靠「三 take + 运行时规则」**：每条情绪 voc 录 **3 个 take**（`_soft` 收敛 / `_mid` 中性 / `_bright` 明亮），由主性格映射选 take，再叠加运行时微调（音高 ±2 半音、时值 ±15%、触发频率）：

| 性格 | take 选择 | 运行时修饰（听感目标） |
|---|---|---|
| 贪吃 | bright | eat 类 voc 频率×1.5、时值+15%——吃得格外投入吧唧 |
| 慵懒 | soft | 全体 -10% 速度、sleepy 类优先——哼唧拖长音 |
| 好奇 | mid | curious 类频率×2，尾音上挑 +1 半音——满脑子问号 |
| 胆小 | soft | 音量 -2dB、首次互动延迟 0.3s 才出声——先观察再回应 |
| 活力 | bright | +8% 速度、happy 类频率×1.5——藏不住的开心 |
| 黏人 | bright | pat 类 voc 后追加一声轻哼（take 复用）——蹭完还要哼 |
| 高冷 | soft | pat 类 30% 概率静默（假装嫌弃）、音量 -2dB——爱你但不说 |
| 淘气 | bright | 玩耍类 +2 半音、节奏切分——得意洋洋 |
| 温柔 | soft | 全体 -1dB、包络更慢起——轻声细语 |
| 爱幻想 | mid | 夜间 curious/sleepy 概率↑、加 0.2s 梦呓尾音——对月亮说话 |

# 10. 动态混音规则（Mixer）

## 10.1 总线结构与并发上限

```
Master（限制器 -0.5dBTP）
 ├─ MUSIC 总线（bgm stems + sting）   用户音量：音乐滑杆
 ├─ AMBIENT 总线（amb，最多同时 3 层） 随音效滑杆 ×0.8
 ├─ SFX 总线（sfx + voc，最多同时 6 声，同 id 最小重触发间隔 80ms）
 └─ UI 总线（ui，最多同时 2 声）
```

- 设置页（S11）：`音乐开关+滑杆`、`音效开关+滑杆`（含 amb/ui/voc）、`震动开关` 三项独立；任一全关后游戏一切照常（对齐零焦虑：声音永远是礼物不是义务）。
- 遵循系统静音开关与音频焦点：来电/其他 App 播放音乐时自动全体淡出 300ms，恢复时淡入 800ms；本 App 不申请打断他人播放（`ambient` audio session category）。

## 10.2 Ducking 规则

| 触发 | 对象 | 幅度 | attack / release |
|---|---|---|---|
| sting_* 播放 | MUSIC(bgm) + AMBIENT | -8 dB | 120ms / 1200ms |
| 收信演出（S8 演出模式） | bgm melody stem | -6 dB（只压旋律，床保留） | 300ms / 2000ms |
| 事件卡手账演出 / 互动动画锁 | bgm melody stem | -4 dB | 300ms / 1500ms |
| voc_* 播放 | AMBIENT | -3 dB | 80ms / 600ms |
| 毕业典礼 S13 | 全体 amb | -12 dB（让 bgm_graduation 独占） | 500ms / 3000ms |

## 10.3 状态过渡与自适应逻辑

- **时段切换（日↔暮↔夜）**：对拍交叉渐变——在当前小节结束点开始 8s 等功率 crossfade 到目标时段曲（三首同 BPM 家族，允许直接混叠）；一次上线会话内最多切 2 次，避免频繁换曲。
- **季节 stem**：随 timeCtx 常开对应季节 stem；换季在次日首次上线时静默生效。
- **豪华度 stems**：st4/st5/st6 按阶段解锁后**从下一个循环小节起淡入（4s）**；豪华度进化演出（sfx_luxury_grow）当次，新 stem 首次亮相与演出同步——「院子长大的那一刻，音乐也长大了」。
- **天气混音配方（不换曲）**：雨天=bgm 全体 -2dB、melody stem -4dB、启用对应 amb_rain；雪天=perc stem 静音、texture +2dB；雷雨=同雨天且 sting/voc 触发频率减半（让世界安静下来陪它躲雨）。
- **回访期**：bgm_revisit_overlay stem 淡入常驻；回访宠离开时随离场演出 6s 淡出。
- **主题皮肤**：切换 amb 主题层 + 混音 EQ 微调预设（如海风主题 MUSIC 高频 +1dB 的「开阔感」）`[待细化：12 主题预设表]`。
- **静音兜底**：连续挂机 20 分钟后 bgm 淡出进入「只剩环境音」的休息态，任意交互淡回——音乐也需要呼吸。
