## 1. 全局美术风格

- **总基调**：手绘水彩、暖色、笨拙可爱、纸质手账质感；整体定位为**奶油 / 可爱风（creamy cute）**。参照《猫咪后院》《旅行青蛙》的温暖手绘感，追求「治愈、松弛、有生活痕迹」，**不要**精致炫酷、高饱和、赛博感。
- **奶油卡通可爱风约束**：所有角色、场景、UI 都应呈现奶油般柔和、圆润、可亲近的观感，同时比纯纸感水彩更鲜亮、更卡通。形体偏圆钝、边缘软，色彩采用**中等饱和的明快奶油色系**：桃粉、嫩绿、天蓝、杏橙、奶油白均衡出现，避免整张图被黄色/米色罩住。可爱来自比例、姿态、材质与细节，不靠荧光色、塑料亮面、夸张 Kawaii 符号或廉价贴纸感。
- **线条**：手绘感线条，允许颜色微微「出线」1–2px，笔触有轻微抖动，不追求几何精确。
- **上色**：水彩晕染、边缘自然渗色、留白透气；叠加淡淡的纸张颗粒纹理（grain），但不得让纸黄覆盖主体色彩。整体应比传统淡水彩更鲜活，保留卡通游戏资产的清晰色块与可读轮廓。
- **光影**：柔和、低对比；避免硬阴影，用暖色环境光统一画面。
- **情绪红线**：一切资产服务于「零焦虑」——不出现尖锐、惊吓、催促感的视觉元素（呼应 DESIGN §0.3）。

### 1.2 渲染质量标准（AAA / App Store 级）★最高优先级硬性红线

> **本节覆盖并作废本文件及所有分域清单（spec-art-*）中任何「SVG 简单图标 / 单色线框 / 占位图 / 原型图 / 草图」的表述。凡冲突，一律以本节为准。**

- **成品级渲染**：所有资产必须是**可直接上架 App Store / Google Play 的终稿美术**，达到一线大厂手游的顶级渲染水准——精致手绘水彩叠加丰富的**光影层次、材质质感、景深、氛围与体积感**，而非扁平简笔。
- **明确禁止**：简单线条/几何 SVG、单色扁平图标、线框、草图、占位图、一眼可辨的廉价/半成品产物。**图标与 UI 元素同样要成品水彩渲染，不得以扁平 SVG 交付。**
- **风格家族**：维持温暖手绘水彩的艺术方向，但以**高保真成品**呈现；完成度对标顶级精致作品（Sky: Children of the Light、Alba、Monument Valley 级的统一与考究）。若需整体转向（写实渲染 / 3D / 其他画风），属**美术方向变更，须另行确认**。
- **一致性兜底**：全项目由第 1 节调色板/线条/质感统一；任何资产不得低于本质量线。
- **交付即终稿**：外部模型/画师产出的每一件都应为**可直接进包的终稿质量**，无需"后期再精修"。附交付前质检：分辨率达标、无锯齿/毛边、透明通道干净、风格与调色板一致、@1x/@2x/@3x 齐全。

## 2. 技术规范

### 2.1 分辨率与导出
- **设计以「逻辑 px」为单位**，交付时导出 **@1x / @2x / @3x** 三套（覆盖 Android/iOS 各屏密度）。清单中标注的尺寸均为 **@1x 逻辑 px**。为顶级渲染留素材余量，背景类源文件建议按 @3x 起绘。
- **全屏满幅、绝不留黑边（第一优先级）**：全屏背景/场景按 **full-bleed 满幅**设计，覆盖最新四代 iPhone 与 Android 的**最宽~最高长宽比**（约 4:3 平板 ~ 21:9 长屏）。做法：画面向四周做**安全溢出（overscan）**，核心内容收在 safe-area 内，边缘为可裁切的延展区——无论何种比例都填满、不出现黑边或拉伸变形。
- **参考画布（权威）**：竖屏 `1290 × 2796`（对齐最新旗舰 iPhone），并提供 safe-area 内容区标注。具体设备矩阵、安全区 inset、响应式锚定规则见 [spec-devices.md](spec-devices.md)。
  - **口径统一**：分域清单（spec-art-world / postcards / ui）早期内文按 `1080×1920` 基准标注的 @1x 尺寸，一律以本参考画布为准**按比例换算 ×1.195**（1290÷1080）；即那些尺寸视为相对值、按此系数放大到 1290 基准。后续会在分域清单中逐步回填为 1290 基准的绝对值 `[待细化]`。
- 卡片/弹层类在 safe-area 内按相对尺寸与锚点摆放，适配不同比例。

### 2.2 文件格式
| 资产类型 | 格式 | 说明 |
|---|---|---|
| 静态图（背景/立绘/贴纸/**图标/UI**） | **PNG**（透明通道，成品水彩渲染） | 全部按终稿质量；**图标/UI 亦为渲染 PNG，不得用扁平 SVG 交付** |
| 高保真源文件 | 分层 **PSD/TIFF**（可选，随 PNG 附交付） | 便于后续调色/复用 |
| 宠物/访客动画 | **Spine / DragonBones 骨骼动画**（首选）或 **序列帧 PNG sprite sheet** | 清单标注帧数与建议帧率 |
| 粒子/天气特效 | 序列帧 或 引擎粒子（Flame）| 标注 |

> SVG 仅可作为**内部矢量草稿/参考**，**不得作为最终交付资产**（见 §1.2 红线）。

### 2.3 动画约定
- 骨骼动画：提供绑定好的部件切图 + 动作列表；序列帧：标注 **帧数 @ fps**（治愈风建议 8–12fps，追求「一顿一顿的笨拙感」）。
- 待机（idle）动画必备呼吸/微动；每个动作 1–2 秒可循环或单次。

### 2.4 命名规范（snake_case）
格式：`<域>_<主体>_<变体/状态>_<细节>`
- 宠物：`pet_cat_var01_stageA_idle`、`pet_shiba_var03_stageD_travel`
- 性格动作：`act_glutton_01`（动作模板，物种适配后加后缀 `_cat`）
- 明信片：`pc_bg_seaside_lighthouse`、`pc_pose_cat_gaze`、`pc_stamp_snowmtn`、`pc_sticker_charsiu`
- 访客：`visitor_fox_portrait`、`visitor_fox_yard`
- 院子：`yard_luxury03_layout`、`yard_theme_sakura`、`deco_water_bowl`
- UI：`ui_btn_feed`、`ui_icon_warmfluff`、`ui_badge_level_stageC`、`ui_tex_paper`

### 2.5 资产条目字段模板（各分域清单统一使用）
| 字段 | 含义 |
|---|---|
| `asset_id / 文件名` | 命名规范下的唯一 ID |
| 用途 | 用在哪个屏幕/系统 |
| 风格与描述 | 画什么、氛围、关键细节（喂给绘图工具的 prompt 基础） |
| 尺寸(@1x 逻辑 px) | 宽×高；动画标画布尺寸 |
| 格式 | PNG / Spine / DragonBones / PNG sprite sheet；SVG 仅可作为内部草稿或参考，不作为最终交付 |
| 帧数·图层 | 动画帧数@fps；分层资产列图层 |
| 变体 | 颜色/状态/物种变体数 |
| 关联 | 对应 content/spec 文档章节 |

## 4. 交付与验收约定
- 每个分域清单结尾附「数量校验表」，核对实际条目数 vs 目标。
- 资产按域建目录：`assets/art/pets/`、`/postcards/`、`/world/`、`/ui/`，文件名 = `asset_id`。
- 风格一致性由本总纲第 1 节兜底；任何分域清单不得引入与调色板/线条约定冲突的风格。
- 动画类资产需附「首帧预览图」供快速核对 `[待细化：预览规格]`。
