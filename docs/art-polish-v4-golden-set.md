## v4 风格判断

- 整体关键词：奶油、可爱、鲜艳、圆润、手账、纸感、水彩厚涂。
- 底色：奶油白和干净纸白为主，不能有整体黄罩、棕罩、旧纸发灰感。
- 彩度：桃粉、珊瑚橙、嫩绿、天蓝、浅紫需要同时出现，避免只靠黄绿或棕橙支撑画面。
- 线条：使用低对比柔棕/珊瑚/蓝灰描边；禁止粗黑线、硬边矢量线。
- 阴影：用柔和接触阴影和水彩晕染，不用强投影、塑料高光、写实体积光。
- 纹理：需要可见纸纤维、轻微水彩边缘、软胶带/剪纸层次；纹理不能脏、花、糊。
- 形状：所有主体尽量圆润、短肢、厚边、低攻击性，轮廓要在小尺寸下可读。

## 分域标准

### 主屏/院子

- 第一眼必须明亮、清爽、可进入；天空和草地不能大面积偏黄。
- 上半区保留 UI 安全区，不能塞满高对比细节。
- 主屏中的宠物应在下三分之一，眼睛和表情清楚。
- 邮箱、食盆、围栏、花朵都要用圆角和奶油高光处理。
- 背景默认不添加额外动物剪影，除非规格明确要求。

### 宠物

- 每个物种先锁 `var01 stageA-D`，再扩展变体和动作。
- 眼睛要大、亮、湿润但不写实；嘴鼻要小，表情友好。
- 成长阶段差异应来自体型、姿势、耳/尾/毛量，而不是换成不同角色。
- 动作帧不能只做平移变形；终稿需要关键帧重绘或骨骼拆层。

### 访客

- 肖像、院内立绘、动作帧要共享同一个轮廓语言。
- 访客可以比宠物更小、更简化，但表情和识别特征必须清楚。
- 院内动作至少要有重心变化、头部/肢体变化，不能只抖动整张图。

### 明信片

- 明信片要像可收藏的小插画，不只是背景占位。
- 地标必须清楚，前中后景分层明确。
- 宠物姿态要自然融入旅行场景，不要像贴上去的普通图鉴头像。
- 默认无文字、无额外动物、无水印；邮票/邮戳可以有图形压印，但不放可读文字。

### UI

- 源 PNG 默认不含可读文字，文字交给引擎叠加。
- 按钮、卡片、徽章、图标都要有纸张层次和柔和接触阴影。
- 图标必须是手绘/印章/贴纸感，不做扁平 SVG 风。
- 后续终稿需要补齐状态：normal、pressed、disabled、selected、unselected。
- @2x/@3x 和 9-patch 切线应在尺寸冻结后统一导出。

## 批量 Polish 检查表

- [ ] 是否比首版更亮、更干净，不偏黄。
- [ ] 是否有桃粉、嫩绿、天蓝、珊瑚橙等多色平衡。
- [ ] 是否保留纸纹和水彩边缘，但没有脏污感。
- [ ] 是否无可读文字、无水印、无错误额外动物。
- [ ] 小尺寸下主体轮廓是否清楚。
- [ ] 同一分域内线条粗细、阴影、饱和度是否一致。
- [ ] 需要透明通道的资产是否有干净 alpha 边缘。
- [ ] 动画/序列帧是否有真实姿态变化，而非单纯位移。
